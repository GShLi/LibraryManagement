## 1. 数据库概述

### 1.1 数据库选型

| 项目 | 说明 |
|------|------|
| 数据库 | SQLite 3 (≥ 3.35) |
| 连接模式 | WAL (Write-Ahead Logging) |
| 字符编码 | UTF-8 |
| 文件位置 | server/data/library.db |
| 备份策略 | 每日全量备份，保留 7 天 |

### 1.2 设计原则

- **第三范式**：消除数据冗余，保证数据一致性
- **外键约束**：开启 SQLite 外键约束（PRAGMA foreign_keys = ON）
- **软删除**：用户表使用 deleted_at 字段而非物理删除
- **审计追溯**：关键操作记录不可删除

## 5. 视图设计

### 5.1 借阅详情视图 (v_borrow_detail)

```sql
CREATE VIEW v_borrow_detail AS
SELECT
    b.id AS borrow_id,
    r.reader_no,
    r.name AS reader_name,
    r.phone AS reader_phone,
    bk.title AS book_title,
    bk.author AS book_author,
    bk.isbn,
    bc.barcode,
    b.borrow_date,
    b.due_date,
    b.return_date,
    b.status AS borrow_status,
    b.renew_count,
    CASE WHEN b.status = 'overdue'
        THEN CAST(julianday('now') - julianday(b.due_date) AS INTEGER)
        ELSE 0
    END AS overdue_days,
    f.amount AS fine_amount,
    f.status AS fine_status
FROM borrow b
JOIN readers r ON b.reader_id = r.user_id
JOIN book_copy bc ON b.copy_id = bc.id
JOIN books bk ON bc.book_id = bk.id
LEFT JOIN fine f ON b.id = f.borrow_id;
```

### 5.2 馆藏统计视图 (v_inventory_stats)

```sql
CREATE VIEW v_inventory_stats AS
SELECT
    bk.id AS book_id,
    bk.title,
    bk.author,
    bk.isbn,
    bk.total_copies,
    bk.available_copies,
    COUNT(CASE WHEN bc.status = 'borrowed' THEN 1 END) AS borrowed_count,
    COUNT(CASE WHEN bc.status = 'withdrawn' THEN 1 END) AS withdrawn_count,
    COUNT(CASE WHEN bc.status = 'stock' THEN 1 END) AS stock_count
FROM books bk
LEFT JOIN book_copy bc ON bk.id = bc.book_id
GROUP BY bk.id;
```
