## 1. 概念模型

### 1.1 实体识别

基于阶段1需求分析，识别出以下核心实体：

| 序号 | 实体 | 英文表名 | 说明 |
|------|------|---------|------|
| 1 | 用户 | users | 系统登录账号，承载认证与角色信息 |
| 2 | 读者 | readers | 读者档案，关联用户，存储借阅相关属性 |
| 3 | 图书 | books | 书目信息（按 ISBN 聚合），描述一种书的元数据 |
| 4 | 图书副本 | book_copy | 单册馆藏，每本实体书对应一条记录，含唯一条码 |
| 5 | 分类 | categories | 中图法分类体系，支持层级树结构 |
| 6 | 借阅记录 | borrow | 每次借出-归还配对的记录 |
| 7 | 罚款记录 | fine | 逾期产生的罚款，与借阅记录一对一 |
| 8 | 预约记录 | reserve | 读者对已借出图书的预约排队 |
| 9 | 操作日志 | audit_log | 关键操作审计追踪 |
| 10 | 系统配置 | system_config | 运行参数 KV 存储 |

### 1.2 实体关系描述

| 关系 | 主体 | 关系类型 | 客体 | 基数 | 说明 |
|------|------|---------|------|------|------|
| R1 | users | 拥有 | readers | 1:1 | 每个登录用户对应一个读者档案；管理员用户可无读者档案 |
| R2 | users | 产生 | audit_log | 1:N | 一个用户可产生多条操作日志 |
| R3 | users | 经手 | borrow | 1:N | 一个操作员(图书管理员)可经手多条借阅(operator_id) |
| R4 | readers | 发起 | borrow | 1:N | 一个读者可有多条借阅记录 |
| R5 | readers | 发起 | reserve | 1:N | 一个读者可预约多本图书 |
| R6 | books | 拥有 | book_copy | 1:N | 一种书可有多个馆藏副本 |
| R7 | books | 关联 | categories | N:1 | 一种书属于一个分类 |
| R8 | books | 被预约 | reserve | 1:N | 一种书可被多次预约 |
| R9 | book_copy | 产生 | borrow | 1:N | 一个副本可被多次借阅（不同时间） |
| R10 | borrow | 产生 | fine | 1:1 | 一条逾期借阅对应一条罚款记录 |
| R11 | categories | 包含 | categories | 1:N | 自引用，支持多级分类树 |

## 3. 逻辑模型

### 3.1 关系模式设计

所有表均满足第三范式（3NF），无传递依赖，无部分依赖。

#### 3.1.1 users — 用户表

**用途**：存储系统登录账号的认证信息和角色。

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PK, AUTOINCREMENT | — | 用户唯一标识 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | — | 登录用户名 |
| password_hash | VARCHAR(255) | NOT NULL | — | bcrypt 加密后的密码哈希 |
| role | VARCHAR(20) | NOT NULL, CHECK(role IN ('admin','librarian','reader')) | 'reader' | 角色：admin=系统管理员, librarian=图书管理员, reader=普通读者 |
| status | VARCHAR(20) | NOT NULL, CHECK(status IN ('active','disabled','locked')) | 'active' | 账户状态 |
| login_attempts | INTEGER | NOT NULL | 0 | 连续登录失败次数，成功后归零 |
| locked_until | DATETIME | NULLABLE | NULL | 锁定到期时间，NULL 表示未锁定 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | — | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | — | 更新时间 |

**范式分析**：
- 主键 id，所有非主属性完全函数依赖主键 → 2NF
- 无传递依赖 → 3NF

**Sequelize 模型名**：`User`

---

#### 3.1.2 readers — 读者表

**用途**：存储读者的个人档案和借阅属性。与 users 表为一对一关系，通过 user_id 外键关联。

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| user_id | INTEGER | PK, FK → users.id | — | 关联用户表 |
| reader_no | VARCHAR(25) | UNIQUE, NOT NULL | — | 读者证号，格式 R-YYYYMMDD-XXXXX |
| name | VARCHAR(50) | NOT NULL | — | 读者姓名 |
| phone | VARCHAR(20) | UNIQUE, NOT NULL | — | 手机号 |
| email | VARCHAR(100) | NULLABLE | NULL | 电子邮箱 |
| reader_type | VARCHAR(20) | NOT NULL, CHECK(reader_type IN ('student','teacher','staff','external')) | 'student' | 读者类型 |
| borrow_limit | INTEGER | NOT NULL | 5 | 最大借阅册数，按读者类型设置默认值 |
| current_borrowed | INTEGER | NOT NULL | 0 | 当前在借数量，借书+1 还书-1 |
| status | VARCHAR(20) | NOT NULL, CHECK(status IN ('active','frozen','lost','disabled')) | 'active' | 读者状态：正常/冻结/挂失/禁用 |
| created_at | DATETIME | NOT NULL | — | 创建时间 |
| updated_at | DATETIME | NOT NULL | — | 更新时间 |

**范式分析**：
- 主键 user_id，所有非主属性完全函数依赖主键 → 2NF
- borrow_limit 由 reader_type 决定（reader_type → borrow_limit），存在轻微冗余——这是刻意保留的性能优化，避免每次借阅时 JOIN system_config。该冗余由应用层保证一致性（修改 reader_type 时同步更新 borrow_limit）。→ 整体满足 3NF

**Sequelize 模型名**：`Reader`

**读者类型默认借阅上限**：

| reader_type | borrow_limit 默认值 |
|-------------|-------------------|
| student | 5 |
| teacher | 10 |
| staff | 5 |
| external | 3 |

---

#### 3.1.3 categories — 分类表

**用途**：存储中图法分类体系，支持层级树结构。

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PK, AUTOINCREMENT | — | 分类唯一标识 |
| code | VARCHAR(20) | UNIQUE, NOT NULL | — | 分类号（如 TP、I、H） |
| name | VARCHAR(100) | NOT NULL | — | 分类名称 |
| parent_id | INTEGER | FK → categories.id, NULLABLE | NULL | 父分类 ID，NULL 表示一级分类 |
| description | VARCHAR(200) | NULLABLE | NULL | 分类说明 |
| created_at | DATETIME | NOT NULL | — | 创建时间 |

**范式分析**：
- 主键 id，自引用外键 parent_id → 3NF

**Sequelize 模型名**：`Category`

**分类体系说明**：中图法（CLC）采用字母+数字编码体系，例如：
- `I` → 文学（一级）
- `TP` → 自动化技术、计算机技术（二级）
- `TP312` → 程序语言、算法语言（三级）

分类表预置中图法 22 个基本大类。二级及以下分类按需扩展。

---

#### 3.1.4 books — 图书表（书目级）

**用途**：按 ISBN 聚合存储图书的元数据。一个 ISBN 对应一条书目记录。

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PK, AUTOINCREMENT | — | 图书唯一标识 |
| isbn | VARCHAR(20) | UNIQUE, NOT NULL | — | ISBN-10 或 ISBN-13 |
| title | VARCHAR(200) | NOT NULL | — | 书名 |
| author | VARCHAR(200) | NOT NULL | — | 作者（多人用逗号分隔） |
| publisher | VARCHAR(100) | NOT NULL | — | 出版社 |
| publish_date | DATE | NOT NULL | — | 出版日期 |
| category_id | INTEGER | FK → categories.id, NOT NULL | — | 关联分类表 |
| price | DECIMAL(10,2) | NOT NULL, CHECK(price >= 0) | — | 定价（元） |
| pages | INTEGER | NULLABLE, CHECK(pages > 0) | NULL | 页数 |
| language | VARCHAR(20) | NOT NULL | '中文' | 语种 |
| edition | VARCHAR(50) | NULLABLE | NULL | 版次（如"第2版"） |
| keywords | TEXT | NULLABLE | NULL | 主题词/标签，JSON 数组格式：["关键词1","关键词2"] |
| description | TEXT | NULLABLE | NULL | 内容简介 |
| cover_url | VARCHAR(500) | NULLABLE | NULL | 封面图片路径（相对路径） |
| status | VARCHAR(20) | NOT NULL, CHECK(status IN ('active','withdrawn')) | 'active' | 书目状态：active=正常 / withdrawn=已全部下架 |
| total_copies | INTEGER | NOT NULL | 0 | 总副本数（冗余字段，由 book_copy 数量汇总） |
| available_copies | INTEGER | NOT NULL | 0 | 可借副本数（冗余字段，由 book_copy 状态汇总） |
| created_at | DATETIME | NOT NULL | — | 入库时间 |
| updated_at | DATETIME | NOT NULL | — | 最后修改时间 |

**范式分析**：
- total_copies 和 available_copies 是冗余字段，由 book_copy 表聚合而来。这是刻意保留的性能优化，避免每次图书检索时 COUNT 子查询或 JOIN。——整体满足 3NF

**Sequelize 模型名**：`Book`

---

#### 3.1.5 book_copy — 图书副本表（册级）

**用途**：存储每一册实体书的馆藏信息。一本图书（1 个 ISBN）可有多个副本。

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PK, AUTOINCREMENT | — | 副本唯一标识 |
| book_id | INTEGER | FK → books.id, NOT NULL | — | 关联图书（书目） |
| barcode | VARCHAR(30) | UNIQUE, NOT NULL | — | 图书条码，格式 BC-YYYYMMDD-XXXXX |
| status | VARCHAR(20) | NOT NULL, CHECK(status IN ('stock','available','borrowed','withdrawn')) | 'stock' | 副本状态 |
| location | VARCHAR(100) | NULLABLE | NULL | 馆藏位置（如"A区-3排-5层"） |
| condition | VARCHAR(20) | NOT NULL, CHECK(condition IN ('new','good','fair','damaged')) | 'new' | 品相 |
| created_at | DATETIME | NOT NULL | — | 入库时间 |
| updated_at | DATETIME | NOT NULL | — | 最后修改时间 |

**范式分析**：
- 主键 id，所有属性完全函数依赖主键 → 3NF

**Sequelize 模型名**：`BookCopy`

**副本状态流转**：

| 状态 | 英文值 | 说明 | 可转换至 |
|------|--------|------|---------|
| 在库 | stock | 初始入库，尚未编目 | available |
| 可借 | available | 编目完成，可被借阅 | borrowed, withdrawn |
| 借出 | borrowed | 当前被借出 | available |
| 已下架 | withdrawn | 下架处理，不可借阅 | available (重新上架) |

---

#### 3.1.6 borrow — 借阅记录表

**用途**：记录每次图书借出、归还、续借的全生命周期数据。

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PK, AUTOINCREMENT | — | 借阅记录唯一标识 |
| reader_id | INTEGER | FK → readers.user_id, NOT NULL | — | 借阅读者 |
| copy_id | INTEGER | FK → book_copy.id, NOT NULL | — | 借出的副本 |
| operator_id | INTEGER | FK → users.id, NOT NULL | — | 操作员（办理借书的图书管理员） |
| borrow_date | DATETIME | NOT NULL | — | 借书日期 |
| due_date | DATETIME | NOT NULL | — | 应还日期 = borrow_date + 借阅天数 |
| return_date | DATETIME | NULLABLE | NULL | 实际归还日期，NULL 表示尚未归还 |
| status | VARCHAR(20) | NOT NULL, CHECK(status IN ('borrowing','returned','overdue')) | 'borrowing' | 借阅状态 |
| renew_count | INTEGER | NOT NULL | 0 | 续借次数，每次续借 +1 |
| created_at | DATETIME | NOT NULL | — | 创建时间 |
| updated_at | DATETIME | NOT NULL | — | 最后修改时间 |

**范式分析**：
- 主键 id → 3NF

**Sequelize 模型名**：`Borrow`

**借阅状态说明**：

| 状态值 | 说明 | 触发条件 |
|--------|------|---------|
| borrowing | 借出中（含续借后） | 借书操作创建；续借后保持此状态 |
| overdue | 逾期未还 | 系统定时任务检测 due_date < NOW() AND return_date IS NULL |
| returned | 已归还 | 还书操作完成 |

---

#### 3.1.7 fine — 罚款记录表

**用途**：记录逾期归还产生的罚款信息。

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PK, AUTOINCREMENT | — | 罚款记录唯一标识 |
| borrow_id | INTEGER | FK → borrow.id, UNIQUE, NOT NULL | — | 关联借阅记录（一对一） |
| overdue_days | INTEGER | NOT NULL, CHECK(overdue_days > 0) | — | 逾期天数 |
| amount | DECIMAL(10,2) | NOT NULL, CHECK(amount > 0) | — | 罚款金额（元） |
| status | VARCHAR(20) | NOT NULL, CHECK(status IN ('unpaid','paid')) | 'unpaid' | 缴纳状态 |
| paid_at | DATETIME | NULLABLE | NULL | 缴纳时间，NULL 表示未缴纳 |
| created_at | DATETIME | NOT NULL | — | 创建时间 |

**范式分析**：
- 主键 id，borrow_id UNIQUE 保证 1:1 → 3NF

**Sequelize 模型名**：`Fine`

**罚款计算规则**：
- 公式：`amount = MIN(overdue_days × 0.1, book.price × 2.0)`
- 费率 0.1 元/天、上限倍数 2.0 均来自 system_config，计算时实时读取

---

#### 3.1.8 reserve — 预约记录表

**用途**：记录读者对已借出图书的预约排队信息。

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PK, AUTOINCREMENT | — | 预约唯一标识 |
| reader_id | INTEGER | FK → readers.user_id, NOT NULL | — | 预约读者 |
| book_id | INTEGER | FK → books.id, NOT NULL | — | 预约的图书（书目级） |
| reserve_date | DATETIME | NOT NULL | — | 预约日期 |
| expire_date | DATETIME | NOT NULL | — | 预约到期日 = reserve_date + 保留天数 |
| status | VARCHAR(20) | NOT NULL, CHECK(status IN ('waiting','fulfilled','cancelled','expired')) | 'waiting' | 预约状态 |
| created_at | DATETIME | NOT NULL | — | 创建时间 |

**范式分析**：
- 主键 id → 3NF

**Sequelize 模型名**：`Reserve`

**预约状态说明**：

| 状态值 | 说明 |
|--------|------|
| waiting | 等待中，图书尚未归还 |
| fulfilled | 已满足，读者已借到该书 |
| cancelled | 读者主动取消 |
| expired | 超期自动失效 |

**预约规则**：
- 仅当图书所有副本均为"已借出"状态时方可预约
- 每位读者同时最多预约 3 本图书（应用层校验）
- 预约保留天数由 system_config.reserve_expire_days 控制（默认 3 天）
- 图书归还后，按预约时间先后（FIFO）通知预约读者

## 5. 索引设计

### 5.1 索引策略

SQLite 默认为主键创建唯一索引。以下为关键查询场景设计额外索引。

**原则**：
- 为所有外键创建索引（加速 JOIN）
- 为高频查询的 WHERE 条件列创建索引
- 为排序/分组列创建索引
- 避免过多索引（影响写入性能）

### 5.3 关键查询 SQL 与索引匹配

#### Q1: 图书多条件检索（高频）

```sql
SELECT b.*, c.name AS category_name,
       COUNT(bc.id) FILTER (WHERE bc.status = 'available') AS avail
FROM books b
LEFT JOIN categories c ON b.category_id = c.id
LEFT JOIN book_copy bc ON b.id = bc.book_id
WHERE (b.title LIKE ? OR b.author LIKE ?)
  AND (? IS NULL OR b.category_id = ?)
  AND (? IS NULL OR b.status = ?)
GROUP BY b.id
ORDER BY b.publish_date DESC
LIMIT 20 OFFSET 0;
```

**索引使用**：idx_books_title (LIKE 前缀), idx_books_author, idx_books_category_id, idx_books_status, idx_books_publish_date (ORDER BY)

#### Q2: 借阅排行（高频）

```sql
SELECT b.id, b.title, b.author, COUNT(br.id) AS borrow_count
FROM borrow br
JOIN book_copy bc ON br.copy_id = bc.id
JOIN books b ON bc.book_id = b.id
WHERE br.borrow_date BETWEEN ? AND ?
GROUP BY b.id
ORDER BY borrow_count DESC
LIMIT 20;
```

**索引使用**：idx_borrow_borrow_date (WHERE 范围), idx_book_copy_book_id (JOIN)

#### Q3: 逾期检测（定时任务，每日执行一次）

```sql
SELECT br.*, r.name, r.phone, bc.barcode, b.title
FROM borrow br
JOIN readers r ON br.reader_id = r.user_id
JOIN book_copy bc ON br.copy_id = bc.id
JOIN books b ON bc.book_id = b.id
WHERE br.status = 'borrowing'
  AND br.due_date < date('now');
```

**索引使用**：idx_borrow_status_due (复合索引，完全覆盖 WHERE 条件)

#### Q4: 还书查找活跃借阅（高频）

```sql
SELECT * FROM borrow
WHERE copy_id = ?
  AND status = 'borrowing'
ORDER BY borrow_date DESC
LIMIT 1;
```

**索引使用**：idx_borrow_copy_id (WHERE), idx_borrow_status (过滤)

#### Q5: 读者借阅历史（高频）

```sql
SELECT br.*, bc.barcode, b.title, b.author
FROM borrow br
JOIN book_copy bc ON br.copy_id = bc.id
JOIN books b ON bc.book_id = b.id
WHERE br.reader_id = ?
ORDER BY br.borrow_date DESC
LIMIT 20 OFFSET ?;
```

**索引使用**：idx_borrow_reader_id (WHERE), idx_borrow_borrow_date (ORDER BY)

### 5.4 索引维护建议

| 维护操作 | 频率 | 说明 |
|---------|------|------|
| 统计信息更新 | 每月 | SQLite: `ANALYZE;` 更新查询优化器统计 |
| 索引重建 | 每季度或数据变更 30% 后 | SQLite: `REINDEX;` |
| 碎片整理 | 每半年 | SQLite: `VACUUM;` 回收空间并重建数据库文件 |

## 7. Sequelize 模型定义要点

### 7.1 模型关联配置

```javascript
// users ↔ readers (1:1)
User.hasOne(Reader, { foreignKey: 'user_id', onDelete: 'CASCADE' });
Reader.belongsTo(User, { foreignKey: 'user_id' });

// categories 自引用 (1:N)
Category.hasMany(Category, { as: 'children', foreignKey: 'parent_id' });
Category.belongsTo(Category, { as: 'parent', foreignKey: 'parent_id' });

// categories → books (1:N)
Category.hasMany(Book, { foreignKey: 'category_id' });
Book.belongsTo(Category, { foreignKey: 'category_id' });

// books → book_copy (1:N)
Book.hasMany(BookCopy, { foreignKey: 'book_id', onDelete: 'CASCADE' });
BookCopy.belongsTo(Book, { foreignKey: 'book_id' });

// readers → borrow (1:N)
Reader.hasMany(Borrow, { foreignKey: 'reader_id' });
Borrow.belongsTo(Reader, { foreignKey: 'reader_id' });

// book_copy → borrow (1:N)
BookCopy.hasMany(Borrow, { foreignKey: 'copy_id' });
Borrow.belongsTo(BookCopy, { foreignKey: 'copy_id' });

// users → borrow (operator) (1:N)
User.hasMany(Borrow, { as: 'operatedBorrows', foreignKey: 'operator_id' });
Borrow.belongsTo(User, { as: 'operator', foreignKey: 'operator_id' });

// borrow → fine (1:1)
Borrow.hasOne(Fine, { foreignKey: 'borrow_id', onDelete: 'CASCADE' });
Fine.belongsTo(Borrow, { foreignKey: 'borrow_id' });

// readers → reserve (1:N)
Reader.hasMany(Reserve, { foreignKey: 'reader_id' });
Reserve.belongsTo(Reader, { foreignKey: 'reader_id' });

// books → reserve (1:N)
Book.hasMany(Reserve, { foreignKey: 'book_id', onDelete: 'CASCADE' });
Reserve.belongsTo(Book, { foreignKey: 'book_id' });

// users → audit_log (1:N)
User.hasMany(AuditLog, { foreignKey: 'user_id' });
AuditLog.belongsTo(User, { foreignKey: 'user_id' });

// users → system_config (1:N)
User.hasMany(SystemConfig, { as: 'updatedConfigs', foreignKey: 'updated_by' });
SystemConfig.belongsTo(User, { as: 'updater', foreignKey: 'updated_by' });
```

### 7.2 SQLite 注意事项

| 事项 | 说明 |
|------|------|
| 外键约束 | SQLite 默认不启用外键约束，需在连接时执行 `PRAGMA foreign_keys = ON;` |
| 并发写入 | SQLite 为文件级锁，仅支持单写者。高并发写入场景需在应用层做排队或使用 WAL 模式 |
| WAL 模式 | 建议启用：`PRAGMA journal_mode=WAL;` 提升读写并发性能 |
| 时间存储 | 统一用 ISO 8601 文本格式（`YYYY-MM-DDTHH:mm:ss.sssZ`），由应用层处理时区 |
| 自增主键 | INTEGER PRIMARY KEY 自动成为 rowid 别名，AUTOINCREMENT 关键字可防止 ID 复用 |
| 迁移 | 使用 Sequelize 的 `sequelize.sync({ alter: true })`（开发阶段）或 Umzug 迁移工具（生产阶段） |
