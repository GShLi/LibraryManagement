## 1. 概念模型

### 1.1 核心实体

| 实体 | 英文名 | 说明 |
|------|--------|------|
| 用户 | User | 系统登录账号，承载认证信息与角色 |
| 读者 | Reader | 读者的个人信息与借阅资质，与 User 为 1:1 扩展关系 |
| 图书 | Book | 书目信息，描述一种书的元数据（ISBN 去重） |
| 图书副本 | BookCopy | 图书的物理馆藏副本，每本一个唯一条码 |
| 分类 | Category | 中图法分类，树形层级结构 |
| 借阅记录 | Borrow | 借出-归还配对的借阅流水 |
| 罚款 | Fine | 逾期归还产生的罚款记录 |
| 预约 | Reserve | 读者对已借出图书的排队预约 |
| 操作日志 | AuditLog | 关键操作审计追踪 |
| 系统配置 | SystemConfig | 系统运行参数键值存储 |

### 1.2 核心关系

| 关系 | 左实体 | 右实体 | 类型 | 基数 | 说明 |
|------|--------|--------|------|------|------|
| 用户—读者 | User | Reader | 扩展 | 1:0..1 | 管理员用户可能无读者档案 |
| 用户—操作日志 | User | AuditLog | 产生 | 1:N | 一个用户可产生多条日志 |
| 读者—借阅记录 | Reader | Borrow | 发起 | 1:N | 一个读者可有多条借阅记录 |
| 用户—借阅记录(操作) | User | Borrow | 经办 | 1:N | 管理员经手多条借阅 |
| 分类—图书 | Category | Book | 归类 | 1:N | 一个分类下有多种图书 |
| 图书—副本 | Book | BookCopy | 拥有 | 1:N | 一种书可有多个物理副本 |
| 副本—借阅记录 | BookCopy | Borrow | 被借 | 1:N | 一个副本可被多次借阅 |
| 借阅记录—罚款 | Borrow | Fine | 衍生 | 1:0..1 | 逾期归还才产生罚款 |
| 读者—预约 | Reader | Reserve | 发起 | 1:N | 一个读者可有多个预约 |
| 图书—预约 | Book | Reserve | 被约 | 1:N | 一种书可有多个预约 |
| 副本—预约 | BookCopy | Reserve | 被约 | 0..1:N | 预约可指定副本或仅指定书目 |

## 3. 逻辑模型（关系模式）

### 3.1 第三范式（3NF）验证

所有关系模式满足：

- **1NF**：所有字段均为原子值，无重复组。`keywords` 字段以 JSON 数组存储，视为单一原子值（SQLite 不要求进一步分解）。
- **2NF**：所有非主键属性完全函数依赖于主键（单列主键天然满足）。
- **3NF**：消除传递依赖。将 `category_name` 从 `books` 表中移出至独立的 `categories` 表，`books` 仅保留 `category_code` 作为外键。

### 3.2 关系模式定义

以下约定适用于所有模式定义：
- PK = PRIMARY KEY, FK = FOREIGN KEY, UQ = UNIQUE, NN = NOT NULL, DF = DEFAULT
- SQLite 整数主键默认 AUTOINCREMENT（使用 INTEGER PRIMARY KEY）
- 时间戳字段统一使用 ISO 8601 格式的 TEXT 类型（SQLite 无原生 DATETIME 类型）

#### 3.2.1 users（用户表）

```
users (id: INTEGER PK, username: TEXT UQ NN, password_hash: TEXT NN,
       role: TEXT NN DF='reader', status: TEXT NN DF='active',
       login_attempts: INTEGER DF=0, locked_until: TEXT,
       deleted_at: TEXT, created_at: TEXT NN, updated_at: TEXT NN)
```

函数依赖：
- id → {username, password_hash, role, status, login_attempts, locked_until, deleted_at, created_at, updated_at}
- username → id

#### 3.2.2 readers（读者表）

```
readers (user_id: INTEGER PK FK→users.id, reader_no: TEXT UQ NN,
         name: TEXT NN, phone: TEXT UQ NN, email: TEXT,
         reader_type: TEXT NN DF='student', borrow_limit: INTEGER NN,
         current_borrowed: INTEGER DF=0, status: TEXT NN DF='active',
         created_at: TEXT NN, updated_at: TEXT NN)
```

函数依赖：
- user_id → {reader_no, name, phone, email, reader_type, borrow_limit, current_borrowed, status, created_at, updated_at}
- reader_no → user_id
- phone → user_id

#### 3.2.3 categories（分类表）

```
categories (id: INTEGER PK, code: TEXT UQ NN, name: TEXT NN,
            parent_code: TEXT FK→categories.code, level: INTEGER NN,
            sort_order: INTEGER DF=0)
```

函数依赖：
- id → {code, name, parent_code, level, sort_order}
- code → {id, name, parent_code, level, sort_order}

说明：自引用外键 `parent_code` 支持树形分类结构（如 TP → 自动化技术是 T → 工业技术的子类）。

#### 3.2.4 books（图书表）

```
books (id: INTEGER PK, isbn: TEXT UQ NN, title: TEXT NN, author: TEXT NN,
       publisher: TEXT NN, publish_date: TEXT NN,
       category_code: TEXT FK→categories.code, price: REAL NN,
       pages: INTEGER, language: TEXT DF='中文', edition: TEXT,
       keywords: TEXT, description: TEXT, cover_url: TEXT,
       status: TEXT NN DF='active', total_copies: INTEGER DF=0,
       available_copies: INTEGER DF=0,
       created_at: TEXT NN, updated_at: TEXT NN)
```

函数依赖：
- id → {isbn, title, author, publisher, publish_date, category_code, price, pages, language, edition, keywords, description, cover_url, status, total_copies, available_copies, created_at, updated_at}
- isbn → id

#### 3.2.5 book_copy（图书副本表）

```
book_copy (id: INTEGER PK, book_id: INTEGER FK→books.id NN,
           barcode: TEXT UQ NN, status: TEXT NN DF='stock',
           location: TEXT, condition: TEXT DF='new',
           created_at: TEXT NN, updated_at: TEXT NN)
```

函数依赖：
- id → {book_id, barcode, status, location, condition, created_at, updated_at}
- barcode → {id, book_id, status, location, condition, created_at, updated_at}

#### 3.2.6 borrow（借阅记录表）

```
borrow (id: INTEGER PK, reader_id: INTEGER FK→readers.user_id NN,
        copy_id: INTEGER FK→book_copy.id NN,
        operator_id: INTEGER FK→users.id NN,
        borrow_date: TEXT NN, due_date: TEXT NN, return_date: TEXT,
        status: TEXT NN DF='borrowing', renew_count: INTEGER DF=0,
        created_at: TEXT NN, updated_at: TEXT NN)
```

函数依赖：
- id → {reader_id, copy_id, operator_id, borrow_date, due_date, return_date, status, renew_count, created_at, updated_at}

#### 3.2.7 fine（罚款表）

```
fine (id: INTEGER PK, borrow_id: INTEGER FK→borrow.id UQ NN,
      overdue_days: INTEGER NN, amount: REAL NN,
      status: TEXT NN DF='unpaid', paid_at: TEXT,
      created_at: TEXT NN)
```

函数依赖：
- id → {borrow_id, overdue_days, amount, status, paid_at, created_at}
- borrow_id → id

#### 3.2.8 reserve（预约表）

```
reserve (id: INTEGER PK, reader_id: INTEGER FK→readers.user_id NN,
         book_id: INTEGER FK→books.id NN,
         copy_id: INTEGER FK→book_copy.id,
         reserve_date: TEXT NN, expire_date: TEXT NN,
         status: TEXT NN DF='waiting',
         created_at: TEXT NN)
```

函数依赖：
- id → {reader_id, book_id, copy_id, reserve_date, expire_date, status, created_at}

说明：`copy_id` 可为 NULL，表示预约该书目的任意可用副本。

#### 3.2.9 audit_log（操作日志表）

```
audit_log (id: INTEGER PK, user_id: INTEGER FK→users.id,
           action: TEXT NN, target_type: TEXT NN,
           target_id: TEXT NN, detail: TEXT,
           ip_address: TEXT NN, created_at: TEXT NN)
```

函数依赖：
- id → {user_id, action, target_type, target_id, detail, ip_address, created_at}

#### 3.2.10 system_config（系统配置表）

```
system_config (id: INTEGER PK, config_key: TEXT UQ NN,
               config_value: TEXT NN, description: TEXT,
               updated_at: TEXT NN, updated_by: INTEGER FK→users.id NN)
```

函数依赖：
- id → {config_key, config_value, description, updated_at, updated_by}
- config_key → id

## 4. 数据字典

| 字段 | 类型 | 约束 | 默认值 | 说明 |

| password_hash | TEXT | NN | — | bcrypt 加盐哈希，不可逆 |
| role | TEXT | NN | 'reader' | admin / librarian / reader |
| status | TEXT | NN | 'active' | active / disabled / locked |
| login_attempts | INTEGER | — | 0 | 连续登录失败计数，成功后归零 |
| locked_until | TEXT | — | NULL | 锁定到期时间（ISO 8601），NULL 表示未锁定 |
| deleted_at | TEXT | — | NULL | 软删除时间戳，NULL 表示正常 |
| created_at | TEXT | NN | — | 创建时间 |
| updated_at | TEXT | NN | — | 最后更新时间 |

### 4.2 readers（读者表）

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| user_id | INTEGER | PK, FK→users.id | — | 读者唯一标识，与登录账号一一对应 |
| reader_no | TEXT | UQ, NN | — | 读者证号，格式 R-YYYYMMDD-XXXXX |
| name | TEXT | NN | — | 读者真实姓名 |
| phone | TEXT | UQ, NN | — | 手机号，1 开头 11 位 |
| email | TEXT | — | NULL | 电子邮箱 |
| reader_type | TEXT | NN | 'student' | student / teacher / staff / external |
| borrow_limit | INTEGER | NN | — | 最大同时借阅册数，按类型设定默认值 |
| current_borrowed | INTEGER | — | 0 | 当前在借数量，借书 +1 还书 -1 |
| status | TEXT | NN | 'active' | active / frozen / lost / disabled |
| created_at | TEXT | NN | — | 注册时间 |
| updated_at | TEXT | NN | — | 最后更新时间 |

### 4.3 categories（分类表）

| 字段 | 类型 | 约束 | 默认值 | 说明 |

## 6. 触发器与约束

### 6.1 检查约束

SQLite 支持 CHECK 约束，以下为建议添加的约束：

```sql
-- users: 角色枚举
CHECK (role IN ('admin', 'librarian', 'reader'))

-- users: 状态枚举
CHECK (status IN ('active', 'disabled', 'locked'))

-- readers: 读者类型枚举
CHECK (reader_type IN ('student', 'teacher', 'staff', 'external'))

-- readers: 读者状态枚举
CHECK (status IN ('active', 'frozen', 'lost', 'disabled'))

-- readers: 借阅计数非负
CHECK (current_borrowed >= 0)

-- books: 书目状态枚举
CHECK (status IN ('active', 'withdrawn'))

-- book_copy: 副本状态枚举
CHECK (status IN ('stock', 'available', 'borrowed', 'withdrawn'))

-- book_copy: 品相枚举
CHECK (condition IN ('new', 'good', 'fair', 'damaged'))

-- borrow: 借阅状态枚举
CHECK (status IN ('borrowing', 'returned', 'overdue'))

-- fine: 罚款状态枚举
CHECK (status IN ('unpaid', 'paid'))

-- reserve: 预约状态枚举
CHECK (status IN ('waiting', 'fulfilled', 'cancelled', 'expired'))
```

### 6.2 外键约束策略

| 外键 | ON DELETE | ON UPDATE | 说明 |
|------|-----------|-----------|------|
| readers.user_id → users.id | CASCADE | CASCADE | 删除用户时级联删除读者档案 |
| books.category_code → categories.code | RESTRICT | CASCADE | 有图书引用时禁止删除分类 |
| book_copy.book_id → books.id | CASCADE | CASCADE | 删除书目时级联删除其所有副本 |
| borrow.reader_id → readers.user_id | RESTRICT | CASCADE | 有借阅记录时禁止删除读者 |

| borrow.operator_id → users.id | RESTRICT | CASCADE | 保留操作员引用 |
| fine.borrow_id → borrow.id | CASCADE | CASCADE | 删除借阅记录时级联删除罚款 |
| reserve.reader_id → readers.user_id | CASCADE | CASCADE | 删除读者时级联删除其预约 |
| reserve.book_id → books.id | CASCADE | CASCADE | 删除书目时级联删除预约 |
| reserve.copy_id → book_copy.id | SET NULL | CASCADE | 删除副本时预约退化为书目级预约 |

## 8. 设计决策说明

### 8.1 为什么将 category_name 从 books 表移出

需求规格说明书将 `category_name` 放在 `books` 表中作为冗余字段。独立 `categories` 表的理由：

1. **消除传递依赖**：`category_name` 函数依赖于 `category_code` 而非 `id`，违反 3NF
2. **数据一致性**：分类名称集中管理，避免同一分类号在多个图书记录中出现不一致的名称
3. **分类层级**：中图法分类具有树形结构（如 T → TP），独立表通过 `parent_code` 自引用支持完整树形表示
4. **API 支持**：需求中 `/api/categories` 端点需要返回分类树，独立表直接对应

### 8.2 为什么保留 total_copies / available_copies / current_borrowed 冗余计数

这些字段是对关联表 COUNT 查询的冗余缓存：

- **性能优化**：避免列表查询时频繁 JOIN + COUNT，尤其是在图书检索（游客高频操作）场景
- **约束校验**：借书时快速判断 `available_copies > 0` 和 `current_borrowed < borrow_limit`
- **维护成本低**：仅在借书、还书、入库、下架四个事务操作时更新，写入频率远低于读取频率

### 8.3 为什么 reserve.copy_id 可为 NULL

预约支持两种粒度：
- **书目级预约**（copy_id IS NULL）：读者预约一种书，任意副本归还后即可满足
- **副本级预约**（copy_id NOT NULL）：读者指定预约某一特定副本

大多数业务场景使用书目级预约；副本级预约保留扩展能力。
