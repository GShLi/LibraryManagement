## 2. 认证模块 /api/auth

### 2.1 用户注册 — POST /api/auth/register

#### 请求定义

**Headers**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Content-Type | string | 是 | application/json |

**Request Body**

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|---------|------|
| username | string | 是 | 3-50 字符，字母开头，仅含字母数字下划线 | 登录用户名 |
| password | string | 是 | 8-32 位，必须包含字母和数字 | 密码 |
| name | string | 是 | 2-50 字符 | 真实姓名 |
| phone | string | 是 | 1 开头 11 位数字，/^1[3-9]\d{9}$/ | 手机号 |
| email | string | 否 | 合法邮箱格式 /^[^\s@]+@[^\s@]+\.[^\s@]+$/ | 电子邮箱 |

**请求示例**

```json
{
  "username": "zhangsan",
  "password": "Abc12345",
  "name": "张三",
  "phone": "13800138000",
  "email": "zhangsan@example.com"
}
```

#### 响应定义

**成功响应 — 201 Created**

```json
{
  "code": 201,
  "message": "注册成功",
  "data": {
    "userId": 3,
    "username": "zhangsan",
    "role": "reader",
    "readerNo": "R-20260704-00003"
  }
}
```

**错误响应**

| 场景 | HTTP | Code | Message |
|------|------|------|---------|
| 用户名已存在 | 409 | 40111 | 用户名已被注册 |
| 手机号已存在 | 409 | 40112 | 手机号已被注册 |
| 密码强度不足 | 422 | 40113 | 密码强度不足，需至少8位且包含字母和数字 |
| 字段校验失败 | 422 | 40000 | 请求参数校验失败 |

```json
{ "code": 40111, "message": "用户名已被注册", "data": null }
```

```json
{ "code": 40112, "message": "手机号已被注册", "data": null }
```

```json
{ "code": 40113, "message": "密码强度不足，需至少8位且包含字母和数字", "data": null }
```

#### 业务规则

**前置条件**
- 无

**校验规则（字段级）**
- username：trim 后长度 3-50，正则 `/^[a-zA-Z][a-zA-Z0-9_]{2,49}$/`
- password：trim 后长度 8-32，正则 `/(?=.*[a-zA-Z])(?=.*\d)/`（必须同时包含字母和数字）
- name：trim 后长度 2-50
- phone：正则 `/^1[3-9]\d{9}$/`
- email（若提供）：正则 `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`

**校验规则（业务级）**
- username 在 users 表（含 deleted_at=NULL）中唯一
- phone 在 readers 表（关联的 users.deleted_at=NULL）中唯一
- email 若提供，在 readers 表中唯一

**后置条件**
- users 表插入一条：role='reader'，status='active'，password bcrypt 哈希，login_attempts=0
- readers 表插入一条：生成读者证号 R-YYYYMMDD-XXXXX（当日自增），reader_type='student'，borrow_limit 按 student 类型取系统配置默认值（默认 5），status='active'
- 记录 audit_log：action='user.register'

**异常处理**
- 数据库插入失败 → 事务回滚，返回 500
- 读者证号生成冲突（极端情况）→ 重试最多 3 次，仍失败返回 500

### 2.2 用户登录 — POST /api/auth/login

#### 请求定义

**Headers**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Content-Type | string | 是 | application/json |

**Request Body**

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|---------|------|
| username | string | 是 | 非空 | 登录用户名 |
| password | string | 是 | 非空 | 密码 |
| rememberMe | boolean | 否 | — | 是否记住我，默认 false |

**请求示例**

```json
{
  "username": "zhangsan",
  "password": "Abc12345",
  "rememberMe": true
}
```

#### 响应定义

**成功响应 — 200 OK**

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 3,
      "username": "zhangsan",
      "role": "reader",
      "name": "张三",
      "readerNo": "R-20260704-00003"
    }
  }
}
```

**错误响应**

| 场景 | HTTP | Code | Message |
|------|------|------|---------|
| 用户名或密码错误 | 401 | 40110 | 用户名或密码错误 |
| 账号已被禁用 | 401 | 40114 | 账号已被禁用，请联系管理员 |
| 账号临时锁定 | 401 | 40115 | 账号已临时锁定，请{minutes}分钟后重试 |
| 字段校验失败 | 422 | 40000 | 请求参数校验失败 |

```json
{ "code": 40110, "message": "用户名或密码错误", "data": null }
```

```json
{ "code": 40114, "message": "账号已被禁用，请联系管理员", "data": null }
```

```json
{ "code": 40115, "message": "账号已临时锁定，请15分钟后重试", "data": null }
```

#### 业务规则

**前置条件**
- 无

**校验规则（字段级）**
- username：非空，trim
- password：非空

**校验规则（业务级）**
- 用户必须存在（users.deleted_at IS NULL）
- 用户状态不能为 disabled
- 若 locked_until 未过期（> now），拒绝登录
- 密码 bcrypt.compare 比对
- 登录失败：login_attempts += 1；若 login_attempts >= login_max_attempts（默认 5），设置 locked_until = now + login_lock_minutes（默认 15 分钟）
- 登录成功：login_attempts 重置为 0，locked_until 置为 NULL

**后置条件**
- 签发 JWT：payload = { userId, username, role }，rememberMe=true 有效期 7 天，否则 24 小时
- 记录 audit_log：action='user.login', target_type='user', target_id=userId
- 登录成功时若 locked_until 已过期，自动解锁

**异常处理**
- JWT 签发失败 → 500
- bcrypt 比对异常 → 500

| password | string | 是 | 8-32 位，含字母和数字 | 初始密码 |
| role | string | 是 | admin / librarian / reader | 角色 |

|------|------|------|---------|

- username 唯一

- users 表插入一条

- 记录 audit_log：action='user.create'，包含操作人信息

**异常处理**
- 事务操作，任一步失败全部回滚
- 读者证号生成冲突重试 3 次

### 3.4 编辑用户信息 — PUT /api/users/:id

#### 请求定义

**Headers**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Authorization | string | 是 | Bearer {token}，需 admin 角色 |

**Path 参数**

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|---------|------|
| id | integer | 是 | > 0 | 用户 ID |

**Request Body**

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|---------|------|
| username | string | 否 | 3-50 字符 | 新用户名 |
| role | string | 否 | admin/librarian/reader | 新角色 |
| name | string | 否 | 2-50 字符 | 读者姓名（仅 reader 角色） |
| phone | string | 否 | 1 开头 11 位数字 | 手机号（仅 reader 角色） |
| email | string | 否 | 合法邮箱格式 | 邮箱（仅 reader 角色） |
| readerType | string | 否 | student/teacher/staff/external | 读者类型 |
| borrowLimit | integer | 否 | >= 1, <= 50 | 借阅上限 |

**请求示例**

```json
{
  "username": "zhangsan_new",
  "name": "张三丰",
  "readerType": "teacher",
  "borrowLimit": 10
}
```

#### 响应定义

**成功响应 — 200 OK**

```json
{ "code": 200, "message": "更新成功", "data": null }
```

**错误响应**

| 场景 | HTTP | Code | Message |
|------|------|------|---------|
| 用户不存在 | 404 | 40201 | 用户不存在 |
| 用户名冲突 | 409 | 40202 | 用户名已存在 |
| 手机号冲突 | 409 | 40112 | 手机号已被注册 |

```json
{ "code": 40202, "message": "用户名已存在", "data": null }
```

#### 业务规则

**前置条件**
- 操作用户角色为 admin
- 目标用户 deleted_at IS NULL

**校验规则（业务级）**
- 若修改 username，新值不能与已有用户冲突（排除自身）
- 若修改 phone，新值不能与已有读者冲突（排除自身）
- 若修改 readerType，自动按配置更新 borrowLimit（除非 borrowLimit 已显式传入）
- 若修改角色（reader → librarian/admin），需校验原读者无未归还借阅记录

**后置条件**
- users 表更新对应字段
- 若为 reader 角色，readers 表更新对应字段
- 记录 audit_log：action='user.update'，记录变更前后的差异

#### 业务规则

**前置条件**
- 操作用户角色为 admin 或 librarian
- readerNo 通过格式识别：R-开头为读者证号精确匹配；纯数字 11 位为手机号

**校验规则（字段级）**
- barcodes 数组非空，最多 20 个元素
- 每个 barcode 非空字符串

**校验规则（业务级）**
- 读者资格校验（逐项检查，遇到第一项失败即返回）：
  1. 读者存在，关联 users.deleted_at IS NULL
  2. readers.status 非 frozen / lost / disabled
  3. 无逾期未还借阅：`SELECT COUNT(*) FROM borrow WHERE reader_id=? AND status IN ('borrowing','overdue') AND due_date < now`
  4. 无未缴罚款：`SELECT COUNT(*) FROM fine WHERE borrow_id IN (SELECT id FROM borrow WHERE reader_id=?) AND status='unpaid'`
  5. 借阅后数量：current_borrowed + barcodes.length <= borrow_limit

- 逐本校验（依次对每个 barcode，遇到问题跳过当前副本继续下一个，全部校验通过后统一事务写入）：
  1. 条码存在
  2. book_copy.status = 'available'（可借）
  3. book_copy 对应的 books.status = 'active'
  4. 检查是否有其他读者对此 book_id 的有效预约（reserve.status='waiting'）
     - 若有预约且读者是借书人本人 → 自动 fulfill 该预约（预留待后续实现）
     - 若有预约但非借书人本人 → 拒绝借出，提示已被预约

- once校验全部通过后，逐个借书事务写入

**后置条件**
- 事务操作（每个条码）：
  1. borrow 表插入一条：reader_id, copy_id, operator_id=req.user.userId, borrow_date=now, due_date=now + borrowDays 天, status='borrowing', renew_count=0
  2. book_copy.status = 'borrowed', book_copy.updated_at = now
  3. readers.current_borrowed += 1
  4. books.available_copies -= 1
  5. 记录 audit_log：action='borrow.create'

**异常处理**
- 并发控制：事务使用 SERIALIZABLE 隔离级别（SQLite 默认串行写）
- 同一副本并发借出 → 先到先得，后者事务冲突回滚返回 409

### 12.6 恢复备份 — POST /api/system/backup/:id/restore

#### 请求定义

**Headers**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Authorization | string | 是 | Bearer {token}，需 admin 角色 |

**Path 参数**

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|---------|------|
| id | integer | 是 | > 0 | 备份记录 ID |

**无 Body 参数**

#### 响应定义

**成功响应 — 200 OK**

```json
{ "code": 200, "message": "备份恢复成功，请重启服务以使恢复生效", "data": null }
```

**错误响应**

| 场景 | HTTP | Code | Message |
|------|------|------|---------|
| 备份不存在 | 404 | 40902 | 备份文件不存在 |
| 恢复失败 | 500 | 40903 | 备份恢复失败 |

```json
{ "code": 40902, "message": "备份文件不存在", "data": null }
```

#### 业务规则

**前置条件**
- 操作用户角色为 admin
- 备份文件存在

**后置条件**
- 用备份文件替换当前 library.db
- 记录 audit_log：action='system.restore'，记录恢复的备份文件名
- **需重启服务**使恢复生效（接口不自动重启）

**异常处理**
- 文件不存在 → 404
- 替换失败（权限、磁盘）→ 500
- **注意**：恢复操作不可逆，建议先创建当前状态备份
