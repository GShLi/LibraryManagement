# 图书管理系统 — 需求描述文档

## 1. 系统概述

### 1.1 项目背景

传统图书管理依赖人工登记、纸质台账的方式进行图书借还和库存管理，存在效率低下、信息查询困难、数据统计不准确等问题。随着数字化管理需求的增长，构建一套轻量级、易部署的图书管理系统，可以有效提升图书馆或机构图书室的运营效率，降低管理成本，改善读者借阅体验。

### 1.2 项目目标

构建一个基于 B/S 架构的图书管理系统，实现以下核心目标：

- **管理数字化**：图书入库、编目、下架全流程线上化，替代纸质台账
- **借阅规范化**：借书、还书、续借、逾期处理标准化操作，减少人工差错
- **服务自助化**：读者可自主检索图书、查看借阅状态、办理续借
- **数据可视化**：提供借阅排行、逾期统计等数据报表，辅助管理决策

### 1.3 项目范围

本系统覆盖以下业务范围：

- 图书全生命周期管理（入库 → 编目 → 上架 → 下架）
- 借阅业务全流程管理（借出 → 归还 → 续借 → 逾期处理）
- 读者信息管理（注册、信息维护、借阅历史查询）
- 多维度查询与统计分析

本系统不包含以下内容：

- 电子书在线阅读功能
- 图书采购/供应商管理
- 财务/费用结算系统
- 移动端 App（仅提供响应式 Web 界面）

### 1.4 用户角色定义

| 角色 | 职责描述 | 典型操作 |
|------|---------|---------|
| 系统管理员 | 系统配置、用户管理、数据维护 | 创建账号、分配权限、系统参数设置、数据备份 |
| 图书管理员 | 图书日常管理、借阅操作处理 | 图书入库/编目/下架、借书/还书、逾期罚款处理 |
| 普通读者 | 图书检索、借阅服务使用 | 检索图书、查看借阅状态、办理续借、查看借阅历史 |
| 游客 | 匿名浏览 | 图书检索浏览、查看图书基本信息 |

## 2. 功能需求

### 2.1 图书管理模块

#### 2.1.1 图书入库

**功能描述**：图书管理员将新购入或捐赠的图书录入系统。

**输入**：
- ISBN（必填，唯一标识）
- 书名（必填）
- 作者（必填）
- 出版社（必填）
- 出版日期（必填）
- 分类号（必填，按中图法分类）
- 定价（必填）
- 页数（选填）
- 内容简介（选填）
- 入库数量（必填，默认为 1）
- 馆藏位置（选填）

**处理逻辑**：
1. 系统校验 ISBN 格式有效性（10 位或 13 位）
2. 校验 ISBN 在系统中是否已存在，若存在则提示合并入库数量
3. 自动生成唯一的图书条码（馆藏号）
4. 自动记录入库时间和操作员信息
5. 图书状态初始化为"在库"
6. 支持批量导入（Excel/CSV 文件上传）

**输出**：入库成功提示及生成的图书条码列表

**异常处理**：
- ISBN 重复：提示已有记录，询问是合并数量还是取消
- ISBN 格式无效：提示格式错误，阻止提交
- 必填字段为空：高亮提示并阻止提交
- 入库数量 ≤ 0：提示"数量必须大于 0"

#### 2.1.2 图书编目

**功能描述**：对已入库的图书进行详细的分类编目，补充完整的书目信息。

**输入**：
- 图书条码（自动关联入库记录）
- 丛书名（选填）
- 版次（选填）
- 语言（默认中文）
- 开本（选填）
- 主题词/关键词（选填，支持多个）
- 附注说明（选填）
- 封面图片（选填，支持上传）

**处理逻辑**：
1. 根据分类号自动匹配分类名称（从预设分类表中查找）
2. 支持编目信息的后续修改
3. 编目完成后图书状态变为"可借"

**输出**：编目完成确认信息

#### 2.1.3 图书下架

**功能描述**：将不再对外借阅的图书标记为下架状态。

**输入**：
- 图书条码（必填）
- 下架原因（必填，选项：遗失、损坏、淘汰、其他）
- 备注说明（选填）

**处理逻辑**：
1. 校验图书是否存在且未被借出（状态为"在库"或"可借"）
2. 如被借出中则提示"该图书正在借出中，不可下架"
3. 更新图书状态为"已下架"
4. 记录下架时间和操作员

**输出**：下架成功确认信息

**异常处理**：
- 图书不存在：提示"未找到该图书"
- 图书已下架：提示"该图书已处于下架状态"
- 图书借出中：提示"该图书正在借出中，不可下架"

### 2.2 借阅管理模块

#### 2.2.1 借书

**功能描述**：图书管理员为读者办理图书借出手续。

**输入**：
- 读者证号/手机号（必填）
- 图书条码（必填，支持同时扫描多本图书）
- 借阅天数（默认 30 天，可手动调整）

**处理逻辑**：
1. 校验读者状态是否正常（未被冻结、无逾期未还图书、借阅数量未达上限）
2. 校验图书状态为"可借"且在架
3. 计算应还日期 = 借书日期 + 借阅天数
4. 创建借阅记录，更新图书状态为"借出"
5. 更新读者当前借阅数量
6. 单次借书数量上限为 5 本

**输出**：借书成功确认，显示应还日期

**异常处理**：
- 读者不存在：提示"未找到该读者"
- 读者被冻结：提示"该读者账户已被冻结，不可借阅"
- 读者有逾期未还：提示"该读者存在逾期未还图书，请先归还"
- 读者借阅数量已满：提示"该读者借阅数量已达上限（5 本）"
- 图书不可借：提示具体原因（已借出/已下架/已预约等）

#### 2.2.2 还书

**功能描述**：图书管理员为读者办理图书归还手续。

**输入**：
- 图书条码（必填）

**处理逻辑**：
1. 根据图书条码查找当前有效的借阅记录
2. 校验是否逾期：实际归还日期 > 应还日期即为逾期
3. 如逾期：计算逾期天数，生成逾期记录，计算罚款金额（0.1 元/天）
4. 更新借阅记录状态为"已归还"，记录实际归还时间
5. 更新图书状态为"可借"
6. 更新读者当前借阅数量

**输出**：还书成功确认，如逾期则显示逾期天数和罚款金额

**异常处理**：
- 图书条码无效：提示"未找到该图书"
- 无有效借阅记录：提示"该图书无有效借阅记录"
- 图书已归还：提示"该图书已归还，无需重复操作"

#### 2.2.3 续借

**功能描述**：读者对已借阅的图书申请延长借阅期限。

**输入**：
- 借阅记录 ID 或图书条码

**处理逻辑**：
1. 校验借阅记录状态为"借出中"
2. 校验续借次数未达上限（最多续借 2 次）
3. 校验该书未被其他读者预约
4. 校验当前无逾期未还图书
5. 新的应还日期 = 原应还日期 + 续借天数（30 天）
6. 续借次数 +1

**输出**：续借成功确认，显示新的应还日期

**异常处理**：
- 续借次数已达上限：提示"续借次数已用完，请归还后重新借阅"
- 已被预约：提示"该图书已被其他读者预约，不可续借"
- 有逾期未还：提示"请先归还逾期图书后再办理续借"

#### 2.2.4 逾期处理

**功能描述**：系统自动检测逾期借阅，提醒管理员和读者进行处理。

**功能子项**：

1. **逾期检测**：
   - 系统每日自动扫描所有"借出中"的借阅记录
   - 将实际日期超过应还日期且未归还的记录标记为"逾期"
   - 生成逾期记录

2. **罚款计算**：
   - 逾期罚款 = 0.1 元/天 × 逾期天数
   - 罚款上限为图书定价的 2 倍
   - 支持管理员手动调整或减免罚款

3. **逾期通知**：
   - 读者登录时弹窗提醒有逾期未还图书
   - 逾期超过 7 天：系统标记读者状态为"警告"
   - 逾期超过 30 天：系统自动冻结读者借阅权限

4. **罚款缴纳**：
   - 还书时自动计算并显示罚款金额
   - 管理员确认收款后标记罚款为"已缴纳"
   - 缴纳完毕后自动解冻读者账户（如因逾期被冻结）

### 2.3 读者管理模块

#### 2.3.1 读者注册

**功能描述**：系统管理员或读者自助注册账号。

**输入**：
- 姓名（必填）
- 手机号（必填，作为登录账号）
- 登录密码（必填，不少于 6 位）
- 邮箱（选填）
- 身份证号（选填）
- 读者类型（必填，选项：学生、教师、职工、外部读者）

**处理逻辑**：
1. 校验手机号格式和唯一性
2. 密码加密存储（bcrypt）
3. 自动生成读者证号（格式：R + 年月日 + 4 位序号）
4. 读者状态初始化为"正常"
5. 默认借阅上限按读者类型设定：学生 5 本、教师 10 本、职工 5 本、外部读者 3 本

**输出**：注册成功，显示读者证号

**异常处理**：
- 手机号已注册：提示"该手机号已注册"
- 手机号格式无效：提示"请输入正确的手机号"
- 密码长度不足：提示"密码不能少于 6 位"
- 必填字段为空：高亮提示并阻止提交

#### 2.3.2 信息维护

**功能描述**：读者或管理员修改读者个人信息。

**可修改字段**：
- 手机号
- 邮箱
- 密码
- 读者类型（仅管理员可修改）
- 借阅上限（仅管理员可修改）

**处理逻辑**：
1. 读者可自助修改部分信息（手机号、邮箱、密码）
2. 管理员可修改所有字段
3. 修改手机号需验证唯一性
4. 修改密码需验证旧密码

**输出**：修改成功确认

#### 2.3.3 借阅历史

**功能描述**：读者或管理员查看历史借阅记录。

**查询条件**：
- 读者证号（必填，读者只能查看自己的）
- 时间范围（选填）
- 借阅状态（选填：全部/借出中/已归还/逾期）

**输出信息**：
- 图书条码、书名、作者
- 借书日期、应还日期、实际归还日期
- 借阅状态（借出中/已归还/逾期）
- 续借次数
- 逾期罚款（如有）

**分页**：每页 20 条记录，按借书日期倒序排列

### 2.4 查询统计模块

#### 2.4.1 图书检索

**功能描述**：支持多条件组合检索图书。

**检索条件**：
- 书名（模糊匹配）
- 作者（模糊匹配）
- ISBN（精确匹配）
- 出版社（模糊匹配）
- 分类号/分类名称（下拉选择）
- 图书状态（可借/借出/已下架/全部）
- 出版年份范围

**输出信息**：
- 图书条码、ISBN、书名、作者、出版社、出版日期
- 分类名称、馆藏位置、当前状态
- 如在借出中则显示预计归还日期

**其他特性**：
- 分页显示（每页 20 条）
- 支持按书名、作者、出版日期排序
- 高级检索：支持多个条件 AND/OR 组合

#### 2.4.2 借阅排行

**功能描述**：统计指定时间段内图书借阅次数排行。

**查询条件**：
- 时间范围（必填，支持按周/月/季度/自定义）
- 分类筛选（选填）
- 排行数量（默认 Top 20）

**输出信息**：
- 排名、图书条码、书名、作者
- 借阅次数
- 当前状态

**展示形式**：排行榜列表 + 简易柱状图（前端实现）

#### 2.4.3 逾期统计

**功能描述**：统计逾期未还的图书和读者信息。

**统计维度**：
- 按时间统计：指定时间段内逾期记录数、逾期率
- 按读者统计：逾期次数最多的读者排行
- 按图书统计：被逾期最多次的图书排行
- 按罚款统计：累计罚款金额统计

**输出信息**：
- 逾期总数、逾期率
- 逾期读者排行（读者证号、姓名、逾期次数、未还数量）
- 逾期图书排行（图书条码、书名、逾期次数）
- 未缴罚款总额

**展示形式**：表格数据 + 统计图表（前端实现）

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | 首屏加载 ≤ 2 秒（3G 网络） |
| 接口响应时间 | 普通查询 ≤ 500ms，复杂统计 ≤ 2s |
| 并发用户数 | 支持 50 个用户同时在线操作 |
| 数据容量 | 支持 10 万册图书、5 万读者、100 万条借阅记录 |
| 批量导入 | 单次支持导入 5000 条图书数据，处理时间 ≤ 30s |

### 3.2 安全需求

- **身份认证**：所有操作需登录后方可进行，游客仅可浏览检索
- **密码安全**：密码使用 bcrypt 加密存储，不以明文传输
- **权限控制**：基于角色的访问控制（RBAC），各角色仅能访问授权功能
- **会话管理**：JWT Token 认证，Token 有效期 24 小时，支持主动登出
- **输入校验**：前后端双重校验，防止 SQL 注入、XSS 攻击
- **操作日志**：记录关键操作（借还、入库、用户创建/删除）的日志，包含操作人、时间、IP 地址
- **数据传输**：生产环境使用 HTTPS 加密传输

### 3.3 可用性需求

- **系统可用性**：7×24 小时可用，计划内维护时间不超过 2 小时/月
- **数据备份**：每日自动备份 SQLite 数据库文件，保留最近 7 天的备份
- **故障恢复**：系统故障后恢复时间目标（RTO）≤ 4 小时
- **容错处理**：接口异常时前端显示友好的错误提示，不暴露系统内部信息
- **数据一致性**：借还操作需保证事务一致性，避免数据不一致

### 3.4 可维护性需求

- **模块化设计**：前后端分离，功能模块解耦，便于独立开发与维护
- **代码规范**：遵循统一的编码规范（ESLint + Prettier）
- **日志记录**：后端使用结构化日志（如 Winston），支持按级别过滤
- **配置外置**：系统配置参数（借阅天数、罚款标准、借阅上限）支持后台动态修改
- **文档完整性**：提供 API 接口文档、数据库设计文档、部署文档

### 3.5 易用性需求

- **界面友好**：使用 Element-Plus 组件库，保持 UI 风格一致
- **操作引导**：关键操作提供提示说明或确认弹窗
- **防错设计**：删除、下架等不可逆操作需二次确认
- **响应式布局**：支持 PC 端（1920×1080 最佳）和平板设备（1024×768 可用）访问
- **键盘快捷操作**：常用功能支持键盘快捷键（如 Enter 提交、Esc 关闭弹窗）

## 4. 约束条件

### 4.1 技术栈约束

| 层级 | 技术选型 | 版本要求 | 说明 |
|------|---------|---------|------|
| 前端框架 | Vue 3 | ≥ 3.3 | 组合式 API + TypeScript |
| UI 组件库 | Element Plus | ≥ 2.3 | 企业级 UI 组件 |
| 前端构建 | Vite | ≥ 4.0 | 快速开发构建 |
| 后端框架 | Express | ≥ 4.18 | Node.js Web 框架 |
| ORM | Sequelize | ≥ 6.0 | 数据库 ORM（支持 SQLite） |
| 认证 | jsonwebtoken | ≥ 9.0 | JWT 认证 |
| 密码加密 | bcryptjs | ≥ 2.0 | 密码哈希 |
| 日志 | winston | ≥ 3.0 | 结构化日志 |
| 数据库 | SQLite 3 | ≥ 3.35 | 轻量级嵌入式数据库 |
| Node.js | — | ≥ 18 LTS | JavaScript 运行时 |

### 4.2 环境约束

- **部署方式**：单机部署，前后端可部署在同一服务器
- **操作系统**：Linux（推荐 Ubuntu 20.04+）或 Windows Server
- **浏览器兼容**：Chrome 90+、Edge 90+、Firefox 90+、Safari 14+
- **网络环境**：局域网或公网均可运行，低带宽环境需保证基本可用
- **硬件要求**：
  - 最低配置：2 核 CPU、4 GB 内存、20 GB 磁盘
  - 推荐配置：4 核 CPU、8 GB 内存、50 GB SSD

### 4.3 业务约束

- 单本图书默认借阅天数为 30 天
- 单次最大借阅数量按读者类型设定（学生 5 本、教师 10 本、职工 5 本、外部读者 3 本）
- 单本图书最多续借 2 次，每次续借 30 天
- 逾期罚款标准：0.1 元/天，上限为图书定价的 2 倍
- 读者逾期超过 30 天自动冻结借阅权限
- 下架图书不可恢复借出，但可查询历史记录
