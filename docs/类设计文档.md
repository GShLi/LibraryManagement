## 1. 设计原则

| 原则 | 说明 |
|------|------|
| 单一职责 | 每个类只有一个变化的原因。Model 只管数据映射，Service 只管业务逻辑，Controller 只管请求处理 |
| 依赖倒置 | Service 依赖 Model 接口（Sequelize Model），不依赖具体数据库实现；可通过配置切换 SQLite/PostgreSQL |
| 分层解耦 | Controller → Service → Model 三层单向依赖，下层不依赖上层 |
| 命名规范 | Model 单数首字母大写(User)，Service 命名 xxxService，Controller 命名 xxxController |
| 错误处理 | Service 层抛出业务异常（AppError），Controller 层通过 try-catch 委托给 errorHandler 中间件 |

#### 2.1.2 User (models/User.js)

```
类名: User
基类: Sequelize.Model
对应表: users

属性:
  - id: INTEGER (PK, AUTO_INCREMENT)         # 用户唯一 ID
  - username: STRING(50) (UNIQUE, NOT NULL)   # 登录用户名
  - password: STRING(255) (NOT NULL)          # bcrypt 加密密码
  - role: ENUM('admin','librarian','reader')  # 角色
  - status: ENUM('active','disabled','locked') # 账号状态
  - login_attempts: INTEGER (DEFAULT: 0)      # 连续登录失败次数
  - locked_until: DATE (NULL)                 # 锁定到期时间
  - created_at: DATE (NOT NULL)
  - updated_at: DATE (NOT NULL)
  - deleted_at: DATE (NULL)                   # 软删除标记

关联:
  - User.hasOne(Reader, { foreignKey: 'user_id' })
  - User.hasMany(Borrow, { foreignKey: 'operator_id', as: 'operatedBorrows' })
  - User.hasMany(AuditLog, { foreignKey: 'user_id' })
  - User.hasMany(SystemConfig, { foreignKey: 'updated_by' })

钩子:
  - beforeCreate: 自动 bcrypt 加密 password，默认 role='reader', status='active'
  - beforeUpdate: 若 password 字段变更则重新 bcrypt 加密
  - beforeBulkUpdate: 同 beforeUpdate

实例方法:
  + validatePassword(password): Promise<bool>  # 验证密码（bcrypt.compare）
  + toSafeJSON(): Object                       # 返回不含 password/login_attempts 的安全对象
  + isLocked(): bool                           # 检查当前是否在锁定期内

查询作用域:
  + defaultScope: { where: { deleted_at: null } }  # 默认排除已软删除的记录
```

### 2.2 服务层 (Services)

#### 2.2.1 AuthService (services/auth.service.js)

```
类名: AuthService
职责: 认证与授权业务逻辑

方法:
  + register({ username, password, name, phone, email }): Promise<{ userId, username, readerNo }>
      1. 校验 username/phone 唯一性
      2. 验证密码强度（≥8位，含字母和数字）
      3. 事务：创建 User + 创建 Reader（自动生成 reader_no）
      4. 记录操作日志
      5. 返回 userId, username, readerNo

  + login({ username, password, rememberMe }): Promise<{ token, user }>
      1. 按 username 查询 User（含软删除检测）
      2. 检查账号状态（disabled/locked）
      3. 检查是否锁定中（locked_until > now）
      4. bcrypt.compare 验证密码
      5. 登录成功：重置 login_attempts=0, 签发 JWT（rememberMe? 7d : 24h）
      6. 登录失败：login_attempts++, 达到阈值锁定 15 分钟
      7. 记录操作日志
      8. 返回 { token, user: {id, username, role, name, readerNo} }

  + getCurrentUser(userId): Promise<User>
      1. 查询 User（含 Reader 关联），排除软删除
      2. 检查账号状态
      3. 返回用户完整信息

  + logout(userId, ip): Promise<void>
      1. 记录登出操作日志
      2. （扩展点：可在此加入 Token 黑名单）

  + validateToken(token): Promise<{ userId, username, role }>
      1. jwt.verify 验证签名和有效期
      2. 返回 Token 载荷

依赖:
  - models: User, Reader
  - utils: jwt, bcrypt, AppError
  - config.auth: JWT 密钥与过期配置
```

```
      4. 记录操作日志

  + updateStatus(id, status, operatorId): Promise<void>
      1. 查询目标 User
      2. 校验状态变更合法性（不能禁用自己）

      4. 同步更新 readers.status
      5. 记录操作日志

  + resetPassword(id, newPassword, operatorId): Promise<void>
      1. 查询目标 User
      2. 设置新密码（触发 beforeUpdate 钩子自动加密）
      3. 记录操作日志

  + changePassword(id, oldPassword, newPassword): Promise<void>
      1. 查询目标 User
      2. 校验 oldPassword
      3. 更新 password
      4. 记录操作日志

  + delete(id, operatorId): Promise<void>
      1. 不能删除自己（抛出 40203）
      2. 检查是否有未归还的借阅记录（reader 角色）
      3. 软删除：设置 deleted_at
      4. 记录操作日志

依赖:
  - models: User, Reader, Borrow
  - utils: bcrypt, AppError
```

#### 2.2.6 BorrowService (services/borrow.service.js)

```
类名: BorrowService
职责: 借阅管理业务逻辑

方法:
  + borrowBooks({ readerNo, barcodes, borrowDays }, operatorId): Promise<{ borrowRecords }>
      1. 根据 readerNo（读者证号或手机号）查询 Reader
      2. 读者资格校验：存在性 + 状态(active) + 借阅上限 + 逾期检查 + 罚款检查
      3. 逐本校验副本：条码存在 + 状态(available) + 预约冲突
      4. 计算应还日期（默认取系统配置 borrow_duration_days）
      5. 事务：
         a. 批量创建 Borrow 记录（每条一本）
         b. 更新 BookCopy.status='borrowed'
         c. 更新 Reader.current_borrowed += N
         d. 更新 Book.available_copies -= N（每条借阅对应不同的书）
         e. 若为预约者本人借书，自动 fulfill 预约
      6. 记录操作日志（每条借阅一条日志）
      7. 返回借阅结果列表

  + returnBook(borrowId, operatorId): Promise<{ borrowId, returnDate, overdue, fine }>
      1. 查询 Borrow 记录（含 BookCopy→Book 关联）
      2. 校验状态为 borrowing 或 overdue（已归还则抛出 40507）
      3. 判断是否逾期：returnDate > due_date
      4. 若逾期：计算逾期天数和罚款金额（含上限校验），创建 Fine 记录
      5. 事务：
         a. 更新 Borrow.return_date + Borrow.status
         b. 更新 BookCopy.status='available'
         c. 更新 Reader.current_borrowed -= 1
         d. 更新 Book.available_copies += 1
         e. 若逾期创建 Fine
         f. 逾期≥30天后自动冻结读者：更新 Reader.status='frozen'
      6. 记录操作日志
      7. 返回还书结果

  + renewBorrow(borrowId, operatorId, requesterRole, requesterId): Promise<{ borrowId, newDueDate, renewCount }>
      1. 查询 Borrow 记录（含 BookCopy→Book 关联）
      2. 权限校验：reader 只能续借自己的
      3. 状态校验：borrowing 状态 + 未逾期 + renew_count < max_renew_count
      4. 预约冲突校验：检查是否有其他读者预约此书
      5. 计算新应还日期 = 原应还日期 + renew_duration_days
      6. 更新 Borrow.due_date + Borrow.renew_count += 1
      7. 记录操作日志
      8. 返回续借结果

  + list({ page, pageSize, status, readerId, startDate, endDate }): Promise<PaginationResult>
      1. 构建 where 条件
      2. 分页查询 Borrow（含 Reader→User, BookCopy→Book, operator→User 关联）
      3. 返回分页结果

  + listOverdue({ page, pageSize, readerId }): Promise<PaginationResult>
      1. where status='overdue' + 可选 readerId
      2. 分页查询（含罚金信息）
      3. 按逾期天数降序排列
      4. 返回分页结果

依赖:
  - models: Borrow, Reader, BookCopy, Book, Fine, Reserve
  - services: SystemConfig (获取借阅/续借/逾期配置)
  - utils: AppError
```

### 2.4 中间件层 (Middlewares)

#### 2.4.1 AuthMiddleware (middleware/auth.js)

```
类名: AuthMiddleware（导出中间件函数）

函数:
  authMiddleware(req, res, next):
      1. 从 req.headers.authorization 提取 Bearer Token
      2. 无 Token 或格式错误 → 抛出 40100: 请先登录
      3. jwt.verify(token, secret)
      4. Token 过期 → 抛出 40101: 登录已过期
      5. Token 无效 → 抛出 40102: 认证失败
      6. 从 Token 载荷提取 { userId, username, role }
      7. 查询 User 确认存在且状态为 active（实时校验，避免 Token 有效但用户已被禁用）
      8. 注入 req.user = { userId, username, role }
      9. next()

  requireAuth(req, res, next):
      若 req.user 不存在 → 抛出 40100

  requireRole(...allowedRoles):
      返回中间件函数 (req, res, next) =>
        若 req.user.role 不在 allowedRoles 中 → 抛出 40300: 权限不足

  optionalAuth(req, res, next):
      有 Token 则解析注入 req.user，无 Token 则 req.user=null 继续

依赖:
  - models: User
  - utils: jwt
```

#### 2.4.4 ErrorHandlerMiddleware (middleware/errorHandler.js)

```
类名: ErrorHandlerMiddleware（导出错误处理中间件）

函数:
  errorHandler(err, req, res, next):
      1. 判断 err 类型：
         - AppError (自定义业务异常):
            返回 { code: err.code, message: err.message, data: err.data }
         - Sequelize.ValidationError (模型校验错误):
            返回 { code: 422, message: "数据校验失败", data: { errors: [...] } }
         - Sequelize.UniqueConstraintError (唯一约束冲突):
            返回 { code: 40900, message: "资源冲突", data: null }
         - JsonWebTokenError / TokenExpiredError:
            返回 { code: 40102, message: "认证失败", data: null }
         - MulterError (文件上传错误):
            返回 { code: 400, message: "文件上传失败", data: null }
         - 其他未知错误:
            记录完整错误栈到 Winston 日志
            返回 { code: 50000, message: "服务器内部错误", data: null }
      2. HTTP 状态码与业务 code 的映射：
         4xx 错误 → 状态码跟随 code 前两位
         50000 → HTTP 500

依赖:
  - utils: AppError
  - logger: Winston
```

#### 3.8.2 Auth Utils (utils/auth.ts)

```
导出函数:

  getToken(): string | null
      return localStorage.getItem('token')

  setToken(token): void
      localStorage.setItem('token', token)

  removeToken(): void
      localStorage.removeItem('token')

  getUser(): UserInfo | null
      return JSON.parse(localStorage.getItem('user') || 'null')

  setUser(user): void
      localStorage.setItem('user', JSON.stringify(user))

  removeUser(): void
      localStorage.removeItem('user')
```

#### 3.8.3 Format Utils (utils/format.ts)

```
导出函数:

  formatDate(date: string | Date, format = 'YYYY-MM-DD'): string
      使用 dayjs 格式化日期

  formatDateTime(date: string | Date): string
      格式化为 YYYY-MM-DD HH:mm:ss

  formatMoney(amount: number): string
      格式化为 ¥X.XX

  formatFileSize(bytes: number): string
      格式化文件大小（KB/MB/GB）

  formatRole(role: string): string
      角色中文名映射: admin→系统管理员, librarian→图书管理员, reader→读者

  formatReaderType(type: string): string
      读者类型中文名: student→学生, teacher→教师, staff→职工, external→外部读者

  formatBookStatus(status: string): string
      图书状态: active→正常, withdrawn→已下架

  formatBorrowStatus(status: string): string
      借阅状态: borrowing→借阅中, returned→已归还, overdue→逾期

  formatReaderStatus(status: string): string
      读者状态: active→正常, frozen→已冻结, lost→已挂失, disabled→已禁用
```
