## 1. 用户认证流程

### 1.1 用户登录时序图

```mermaid
sequenceDiagram
    actor User as 用户
    participant Vue as Vue3前端
    participant Axios as Axios拦截器
    participant Express as Express服务
    participant AuthMw as Auth中间件
    participant AuthCtrl as AuthController
    participant AuthSvc as AuthService
    participant DB as SQLite (Sequelize)

    User->>Vue: 输入用户名/密码，点击登录
    Vue->>Vue: Element Plus 表单校验
    alt 前端校验失败
        Vue-->>User: 显示字段校验提示
    end

    Vue->>Axios: POST /api/auth/login {username, password, rememberMe}
    Axios->>Express: HTTP POST (无需 Token)
    Note over Express: CORS → Helmet → BodyParser → Logger

    Express->>AuthCtrl: login(req, res)
    AuthCtrl->>AuthSvc: login(username, password, rememberMe)

    AuthSvc->>DB: users.findOne({where:{username}})
    DB-->>AuthSvc: user 记录 或 null

    alt 用户不存在
        AuthSvc-->>AuthCtrl: throw AuthError(40110, "用户名或密码错误")
    end

    AuthSvc->>AuthSvc: 检查 user.status
    alt status=disabled
        AuthSvc-->>AuthCtrl: throw AuthError(40114, "账号已被禁用")
    end
    alt status=locked 且 lockedUntil > now
        AuthSvc-->>AuthCtrl: throw AuthError(40115, "账号已临时锁定，请15分钟后重试")
    end

    AuthSvc->>AuthSvc: bcrypt.compare(password, user.password)
    alt 密码错误
        AuthSvc->>DB: users.update({loginAttempts: +1}, {where:{id}})
        alt loginAttempts >= 5
            AuthSvc->>DB: users.update({status:'locked', lockedUntil: now+15min})
        end
        AuthSvc-->>AuthCtrl: throw AuthError(40110, "用户名或密码错误")
    end

    AuthSvc->>DB: users.update({loginAttempts:0, lockedUntil:null})
    AuthSvc->>AuthSvc: 生成 JWT Token
    Note over AuthSvc: payload: {userId, username, role}<br/>HS256签名, rememberMe?7d:24h
    AuthSvc->>DB: audit_log.create({action:'user.login', ...})
    AuthSvc-->>AuthCtrl: {token, user:{id,username,role,name,readerNo}}
    AuthCtrl-->>Axios: {code:200, message:"登录成功", data:{token, user}}

    Axios->>Axios: localStorage.setItem('token', token)
    Axios-->>Vue: 登录成功响应
    Vue->>Vue: router.push(对应角色首页)
    Vue-->>User: 跳转到首页

    Note over User,Vue: 登录失败流程
```

### 1.2 Token 校验与刷新时序图

```mermaid
sequenceDiagram
    actor User as 用户
    participant Vue as Vue3前端
    participant Router as Vue Router
    participant Axios as Axios拦截器
    participant Express as Express服务
    participant AuthMw as Auth中间件(JWT)
    participant RBAC as RBAC中间件
    participant Controller as Controller
    participant Service as Service
    participant DB as SQLite

    Note over User,DB: Token 正常流程
    User->>Vue: 发起页面访问或操作
    Vue->>Axios: API请求(GET/POST/PUT/DELETE)
    Axios->>Axios: 从 localStorage 读取 token
    alt token 不存在
        Axios->>Vue: 返回 401
        Vue->>Router: 跳转登录页 /login
    end
    Axios->>Express: 附带 Header: Authorization: Bearer <token>

    Express->>AuthMw: auth(req, res, next)
    AuthMw->>AuthMw: 提取 Bearer Token
    alt Token 缺失
        AuthMw-->>Express: 401 {code:40100, message:"请先登录"}
        Express-->>Axios: 401 响应
    end

    AuthMw->>AuthMw: jwt.verify(token, JWT_SECRET)
    alt Token 过期
        AuthMw-->>Express: 401 {code:40101, message:"登录已过期，请重新登录"}
        Express-->>Axios: 401 响应
        Axios->>Axios: localStorage.removeItem('token')
        Axios->>Vue: 触发 401 处理
        Vue->>Router: 跳转登录页 /login
    end
    alt Token 签名无效
        AuthMw-->>Express: 401 {code:40102, message:"认证失败"}
    end

    AuthMw->>AuthMw: 注入 req.user = {id, username, role}
    AuthMw->>RBAC: next()

    Note over User,DB: RBAC 权限校验
    RBAC->>RBAC: 根据路由配置检查 req.user.role
    alt 角色权限不足
        RBAC-->>Express: 403 {code:40300, message:"权限不足"}
    end
    RBAC->>Controller: next()

    Note over Controller,DB: 后续正常业务处理
    Controller->>Service: 调用业务方法
    Service->>DB: 数据操作
    DB-->>Service: 操作结果
    Service-->>Controller: 业务结果
    Controller-->>Express: {code:200, message:"ok", data:{...}}
    Express-->>Axios: 200 响应
    Axios-->>Vue: 数据返回

    Note over User,DB: Token 过期前端处理流程
```

## 2. 图书管理流程

### 2.1 图书入库时序图

```mermaid
sequenceDiagram
    actor Admin as 图书管理员
    participant Vue as Vue3前端
    participant Axios as Axios拦截器
    participant Express as Express服务
    participant AuthMw as JWT中间件
    participant RBAC as RBAC中间件
    participant BookCtrl as BookController
    participant BookSvc as BookService
    participant DB as SQLite (Sequelize)

    Admin->>Vue: 填写图书信息，点击入库
    Vue->>Vue: Element Plus 表单校验
    Note over Vue: ISBN格式、必填字段、copyCount>0

    alt 前端校验失败
        Vue-->>Admin: 高亮错误字段提示
    end

    Vue->>Axios: POST /api/books {isbn,title,author,publisher,...copyCount}
    Axios->>Express: 附带 JWT Token

    Express->>AuthMw: JWT 认证
    AuthMw-->>Express: req.user = {id, username, role}
    Express->>RBAC: role in [admin, librarian]
    alt 权限不足
        RBAC-->>Express: 403
        Express-->>Axios: 403 响应
        Axios-->>Vue: 提示"权限不足"
    end

    Express->>BookCtrl: createBook(req, res)
    BookCtrl->>BookSvc: importBook(bookData, operatorId)

    BookSvc->>BookSvc: 校验ISBN格式(10/13位)
    alt ISBN格式错误
        BookSvc-->>BookCtrl: throw 40406 "ISBN格式错误"
    end

    BookSvc->>DB: books.findOne({where:{isbn}})
    DB-->>BookSvc: 已存在记录 或 null

    alt ISBN已存在
        BookSvc-->>BookCtrl: throw 40402 "ISBN已存在，请选择增加复本"
    end

    alt copyCount无效(≤0)
        BookSvc-->>BookCtrl: throw 40409 "入库数量无效"
    end

    BookSvc->>DB: sequelize.transaction()
    Note over BookSvc,DB: 事务开始

    BookSvc->>DB: books.create({isbn,title,author,...,totalCopies:copyCount,availableCopies:copyCount})
    DB-->>BookSvc: newBook {id, ...}

    loop copyCount 次
        BookSvc->>BookSvc: 生成条码 BC-YYYYMMDD-XXXXX
        BookSvc->>DB: book_copy.create({bookId, barcode, status:'stock', location})
    end
    DB-->>BookSvc: barcodes[]

    BookSvc->>DB: audit_log.create({action:'book.import', ...})
    BookSvc->>DB: commit()

    Note over BookSvc,DB: 事务提交

    BookSvc-->>BookCtrl: {bookId, isbn, totalCopies, barcodes}
    BookCtrl-->>Express: {code:201, message:"入库成功", data:{...}}
    Express-->>Axios: 201 响应
    Axios-->>Vue: 入库成功
    Vue-->>Admin: 显示入库成功 + 条码列表

    Note over Admin,DB: 并发冲突: 事务内由SQLite行锁保证<br/>异常: 事务自动回滚
```

## 3. 借阅管理流程

### 3.1 借书时序图（核心流程）

```mermaid
sequenceDiagram
    actor Admin as 图书管理员
    participant Vue as Vue3前端
    participant Axios as Axios拦截器
    participant Express as Express服务
    participant AuthMw as JWT中间件
    participant RBAC as RBAC中间件
    participant BorrowCtrl as BorrowController
    participant BorrowSvc as BorrowService
    participant ReserveSvc as ReserveService
    participant DB as SQLite (Sequelize)

    Admin->>Vue: 输入读者证号(或扫描) + 扫描图书条码
    Vue->>Vue: 前端验证格式

    Vue->>Axios: POST /api/borrows {readerNo, barcodes:[...]}
    Axios->>Express: 附带 JWT Token

    Express->>AuthMw: JWT 认证
    AuthMw-->>Express: req.user = {id,username,role}
    Express->>RBAC: role in [admin, librarian]
    RBAC-->>Express: 放行

    Express->>BorrowCtrl: createBorrow(req, res)
    BorrowCtrl->>BorrowSvc: borrowBooks(readerNo, barcodes, operatorId)

    Note over BorrowSvc,DB: === 阶段1: 读者资格校验 ===

    BorrowSvc->>DB: readers.findOne({where: {readerNo} or {phone:readerNo}})
    DB-->>BorrowSvc: reader 或 null

    alt 读者不存在
        BorrowSvc-->>BorrowCtrl: throw 40301 "读者不存在"
    end

    BorrowSvc->>BorrowSvc: 校验 reader.status
    alt reader.status=frozen
        BorrowSvc-->>BorrowCtrl: throw 40302 "读者账户已冻结"
    end
    alt reader.status=lost
        BorrowSvc-->>BorrowCtrl: throw 40303 "读者账户已挂失"
    end
    alt reader.status=disabled
        BorrowSvc-->>BorrowCtrl: throw 40304 "读者账户已禁用"
    end

    opt 有未还逾期图书
        BorrowSvc->>DB: borrow.count({where:{readerId, status:'overdue'}})
        DB-->>BorrowSvc: overdueCount
        alt overdueCount > 0
            BorrowSvc-->>BorrowCtrl: throw 40503 "读者有逾期未还图书"
        end
    end

    opt 有未缴罚款
        BorrowSvc->>DB: fine.count({where:{status:'unpaid'}, include:[{model:Borrow, where:{readerId}}]})
        DB-->>BorrowSvc: unpaidFineCount
        alt unpaidFineCount > 0
            BorrowSvc-->>BorrowCtrl: throw 40504 "读者有未缴纳罚款"
        end
    end

    alt reader.currentBorrowed + barcodes.length > reader.borrowLimit
        BorrowSvc-->>BorrowCtrl: throw 40502 "读者已达借阅上限"
    end

    Note over BorrowSvc,DB: === 阶段2: 逐本校验副本状态 ===

    loop barcodes 中每条
        BorrowSvc->>DB: book_copy.findOne({where:{barcode}, include:[Book]})
        DB-->>BorrowSvc: copy + book

        alt 条码不存在
            BorrowSvc-->>BorrowCtrl: throw 40407 "副本不存在: {barcode}"
        end
        alt copy.status != 'available'
            BorrowSvc-->>BorrowCtrl: throw 40408 "该副本不可借: {barcode}"
        end
        alt book.status = 'withdrawn'
            BorrowSvc-->>BorrowCtrl: throw 40405 "该图书已下架"
        end

        opt 检查预约冲突
            BorrowSvc->>ReserveSvc: checkReserveConflict(bookId, readerId)
            ReserveSvc->>DB: reserve.findOne({where:{bookId,status:'waiting', readerIdNot}})
            DB-->>ReserveSvc: conflictReserve
            alt 有其他读者预约
                ReserveSvc-->>BorrowSvc: {conflict:true}
                BorrowSvc-->>BorrowCtrl: throw 40506 "该图书已被其他读者预约"
            end
        end
    end

    Note over BorrowSvc,DB: === 阶段3: 事务操作 ===

    BorrowSvc->>DB: sequelize.transaction()

    loop barcodes 中每条
        BorrowSvc->>BorrowSvc: 计算 dueDate = now + borrowDurationDays
        BorrowSvc->>DB: borrow.create({readerId,copyId,operatorId,borrowDate,dueDate,status:'borrowing'})
        BorrowSvc->>DB: book_copy.update({status:'borrowed'}, {where:{id:copyId}})
    end

    BorrowSvc->>DB: readers.update({currentBorrowed: +N}, {where:{userId:readerId}})

    loop 涉及的每本图书
        BorrowSvc->>DB: books.update({availableCopies: availableCopies-N}, {where:{id:bookId}})
        Note over DB: 冗余字段同步更新
    end

    opt 读者是预约者 → 自动 fulfill 预约
        loop 借出的每本书
            BorrowSvc->>ReserveSvc: fulfillReserve(bookId, readerId)
            ReserveSvc->>DB: reserve.update({status:'fulfilled'}, {where:{bookId,readerId,status:'waiting'}})
        end
    end

    BorrowSvc->>DB: audit_log.create({action:'borrow.create', ...})
    BorrowSvc->>DB: commit()

    Note over BorrowSvc,DB: 事务提交

    BorrowSvc-->>BorrowCtrl: {borrowRecords:[{borrowId,barcode,bookTitle,dueDate},...]}
    BorrowCtrl-->>Express: {code:201, message:"借书成功", data:{borrowRecords}}
    Express-->>Axios: 201 响应
    Axios-->>Vue: 借书成功
    Vue-->>Admin: 显示借阅成功 + 应还日期列表

    Note over Admin,DB: 并发冲突: 同一副本被同时借出时<br/>SQLite事务行锁导致第二个事务等待/失败<br/>→回滚返回 409 "该图书已被借出"
```

### 3.2 还书时序图

```mermaid
sequenceDiagram
    actor Admin as 图书管理员
    participant Vue as Vue3前端
    participant Axios as Axios拦截器
    participant Express as Express服务
    participant AuthMw as JWT中间件
    participant RBAC as RBAC中间件
    participant BorrowCtrl as BorrowController
    participant BorrowSvc as BorrowService
    participant FineSvc as FineService
    participant DB as SQLite (Sequelize)

    Admin->>Vue: 扫描图书条码，点击还书
    Vue->>Axios: PUT /api/borrows/:id/return
    Axios->>Express: 附带 JWT Token

    Express->>AuthMw: JWT 认证
    AuthMw-->>Express: req.user
    Express->>RBAC: role in [admin, librarian]
    RBAC-->>Express: 放行

    Express->>BorrowCtrl: returnBook(req, res)
    BorrowCtrl->>BorrowSvc: returnBook(borrowId)

    Note over BorrowSvc,DB: === 阶段1: 查找借阅记录 ===

    BorrowSvc->>DB: borrow.findOne({where:{id:borrowId}, include:[CopyBook,Reader]})
    DB-->>BorrowSvc: borrow 记录 或 null

    alt 借阅记录不存在
        BorrowSvc-->>BorrowCtrl: throw 40501 "借阅记录不存在"
    end

    alt borrow.status = 'returned'
        BorrowSvc-->>BorrowCtrl: throw 40507 "该记录已归还"
    end

    Note over BorrowSvc,DB: === 阶段2: 计算逾期 ===

    BorrowSvc->>BorrowSvc: returnDate = new Date()
    BorrowSvc->>BorrowSvc: isOverdue = returnDate > dueDate

    alt 逾期
        BorrowSvc->>BorrowSvc: overdueDays = floor((returnDate-dueDate)/86400000)
        BorrowSvc->>BorrowSvc: fineAmount = overdueDays × 0.1
        BorrowSvc->>DB: system_config.findOne({where:{config_key:'fine_max_multiple'}})
        DB-->>BorrowSvc: fineMaxMultiple
        BorrowSvc->>DB: books.findOne({where:{id: borrow.copy.book.id}})
        DB-->>BorrowSvc: bookPrice
        BorrowSvc->>BorrowSvc: fineAmount = min(fineAmount, bookPrice×fineMaxMultiple)
    end

    Note over BorrowSvc,DB: === 阶段3: 事务操作 ===

    BorrowSvc->>DB: sequelize.transaction()

    alt 逾期
        BorrowSvc->>DB: borrow.update({status:'returned', returnDate}, {where:{id:borrowId}})
        BorrowSvc->>DB: fine.create({borrowId, overdueDays, amount:fineAmount, status:'unpaid'})
    else 正常归还
        BorrowSvc->>DB: borrow.update({status:'returned', returnDate}, {where:{id:borrowId}})
    end

    BorrowSvc->>DB: book_copy.update({status:'available'}, {where:{id: borrow.copyId}})
    BorrowSvc->>DB: readers.update({currentBorrowed: currentBorrowed-1}, {where:{userId:borrow.readerId}})
    BorrowSvc->>DB: books.update({availableCopies: availableCopies+1}, {where:{id:borrow.copy.bookId}})

    alt 逾期天数 ≥ 30 且读者状态正常
        BorrowSvc->>DB: readers.update({status:'frozen'}, {where:{userId:borrow.readerId, status:'active'}})
        Note over DB: 自动冻结读者账户
    end

    BorrowSvc->>DB: audit_log.create({action:'borrow.return', ...})
    BorrowSvc->>DB: commit()

    BorrowSvc-->>BorrowCtrl: {borrowId, returnDate, overdue:isOverdue, ...fine}
    BorrowCtrl-->>Express: 还书结果

    alt 正常归还
        Express-->>Axios: {code:200, message:"还书成功", data:{overdue:false}}
    else 逾期归还
        Express-->>Axios: {code:200, message:"还书成功，该图书已逾期，请缴纳罚款", data:{overdue:true, overdueDays, fine:{fineId,amount,status:'unpaid'}}}
    end

    Axios-->>Vue: 还书结果
    Vue-->>Admin: 显示还书结果 + 逾期罚款信息(如有)
```

### 3.3 续借时序图

```mermaid
sequenceDiagram
    actor Reader as 读者
    participant Vue as Vue3前端
    participant Axios as Axios拦截器
    participant Express as Express服务
    participant AuthMw as JWT中间件
    participant RBAC as RBAC中间件
    participant BorrowCtrl as BorrowController
    participant BorrowSvc as BorrowService
    participant ReserveSvc as ReserveService
    participant DB as SQLite (Sequelize)

    Reader->>Vue: 在个人中心点击"续借"
    Vue->>Axios: PUT /api/borrows/:id/renew
    Axios->>Express: 附带 JWT Token

    Express->>AuthMw: JWT 认证
    AuthMw-->>Express: req.user = {id, username, role}
    Express->>RBAC: role in [reader(own), admin, librarian]
    RBAC-->>Express: 放行

    Express->>BorrowCtrl: renewBook(req, res)
    BorrowCtrl->>BorrowSvc: renewBook(borrowId, userId)

    Note over BorrowSvc,DB: === 阶段1: 校验借阅记录 ===

    BorrowSvc->>DB: borrow.findOne({where:{id:borrowId}, include:[CopyBook]})
    DB-->>BorrowSvc: borrow 记录

    alt 借阅记录不存在
        BorrowSvc-->>BorrowCtrl: throw 40501 "借阅记录不存在"
    end

    alt borrow.readerId != userId (读者只能续借自己的)
        BorrowSvc-->>BorrowCtrl: throw 40300 "权限不足"
    end

    alt borrow.status = 'overdue'
        BorrowSvc-->>BorrowCtrl: throw 40508 "逾期图书不可续借，请先归还"
    end
    alt borrow.status != 'borrowing'
        BorrowSvc-->>BorrowCtrl: throw 40509 "该记录状态不可续借"
    end

    Note over BorrowSvc,DB: === 阶段2: 校验续借条件 ===

    BorrowSvc->>DB: system_config.findOne({where:{config_key:'max_renew_count'}})
    DB-->>BorrowSvc: maxRenewCount

    alt borrow.renewCount >= maxRenewCount
        BorrowSvc-->>BorrowCtrl: throw 40509 "续借次数已用完"
    end

    BorrowSvc->>ReserveSvc: checkReserveConflict(borrow.bookId, userId)
    ReserveSvc->>DB: reserve.findOne({where:{bookId:borrow.copy.bookId, status:'waiting', readerId:{$ne:userId}}})
    DB-->>ReserveSvc: conflictReserve

    alt 有其他读者预约
        ReserveSvc-->>BorrowSvc: {conflict:true}
        BorrowSvc-->>BorrowCtrl: throw 40510 "该图书已被其他读者预约，不可续借"
    end

    Note over BorrowSvc,DB: === 阶段3: 执行续借 ===

    BorrowSvc->>DB: system_config.findOne({where:{config_key:'renew_duration_days'}})
    DB-->>BorrowSvc: renewDurationDays

    BorrowSvc->>BorrowSvc: newDueDate = borrow.dueDate + renewDurationDays (天数)
    BorrowSvc->>BorrowSvc: newRenewCount = borrow.renewCount + 1

    BorrowSvc->>DB: sequelize.transaction()
    BorrowSvc->>DB: borrow.update({dueDate:newDueDate, renewCount:newRenewCount}, {where:{id:borrowId}})
    BorrowSvc->>DB: audit_log.create({action:'borrow.renew', ...})
    BorrowSvc->>DB: commit()

    BorrowSvc-->>BorrowCtrl: {borrowId, newDueDate, renewCount:newRenewCount}
    BorrowCtrl-->>Express: {code:200, message:"续借成功", data:{newDueDate, renewCount}}
    Express-->>Axios: 200 响应
    Axios-->>Vue: 续借成功
    Vue-->>Reader: 显示新应还日期 + 续借次数

    Note over Reader,DB: 续借不改变 borrow.status (仍为 borrowing)
```

## 4. 逾期处理流程

### 4.1 逾期检测与罚款生成时序图（定时任务）

```mermaid
sequenceDiagram
    participant Cron as 定时任务(node-cron)
    participant App as Express应用
    participant OverdueJob as OverdueJob
    participant BorrowSvc as BorrowService
    participant FineSvc as FineService
    participant SystemSvc as SystemService
    participant DB as SQLite

    Note over Cron: 每日 00:00 触发

    Cron->>OverdueJob: overdueCheck()

    Note over OverdueJob,DB: === 阶段1: 标记逾期记录 ===

    OverdueJob->>DB: borrow.updateMany({status:'overdue'}, {where:{status:'borrowing', dueDate:{$lt:new Date()}}})
    DB-->>OverdueJob: affectedRows (新标记为逾期的记录数)

    alt affectedRows = 0
        OverdueJob-->>Cron: 无新逾期记录，任务结束
    end

    Note over OverdueJob,DB: === 阶段2: 为逾期记录自动创建罚款 ===

    OverdueJob->>DB: borrow.findAll({where:{status:'overdue', dueDate:{$lt:new Date()}}, include:[CopyBook]})
    Note over DB: 查询逾期且未创建罚款的记录
    DB-->>OverdueJob: overdueBorrows[]

    OverdueJob->>DB: system_config.findOne({where:{config_key:'overdue_fine_rate'}})
    DB-->>OverdueJob: fineRate (默认0.1元/天)

    OverdueJob->>DB: system_config.findOne({where:{config_key:'fine_max_multiple'}})
    DB-->>OverdueJob: fineMaxMultiple (默认2.0倍)

    OverdueJob->>DB: system_config.findOne({where:{config_key:'overdue_freeze_days'}})
    DB-->>OverdueJob: freezeDays (默认30天)

    loop overdueBorrows 中每条
        OverdueJob->>OverdueJob: overdueDays = floor((now - dueDate) / 86400000)
        OverdueJob->>OverdueJob: maxFine = book.price × fineMaxMultiple
        OverdueJob->>OverdueJob: fineAmount = min(overdueDays × fineRate, maxFine)

        OverdueJob->>DB: sequelize.transaction()

        OverdueJob->>DB: fine.create({borrowId, overdueDays, amount:fineAmount, status:'unpaid'})

        alt overdueDays >= freezeDays
            OverdueJob->>DB: readers.findOne({where:{userId:borrow.readerId}})
            DB-->>OverdueJob: reader
            alt reader.status = 'active'
                OverdueJob->>DB: readers.update({status:'frozen'}, {where:{userId:borrow.readerId}})
                Note over DB: 自动冻结读者账户
            end
        end

        OverdueJob->>DB: commit()
    end

    OverdueJob->>DB: audit_log.create({action:'system.overdue_check', target_type:'borrow', detail:{affectedRows, fineCount}})

    OverdueJob-->>Cron: overdueCheck完成
    Cron-->>Cron: 记录执行日志

    Note over Cron,DB: 定时任务不涉及用户交互<br/>所有操作在后端静默执行
```
