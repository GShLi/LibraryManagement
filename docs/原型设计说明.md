## 1. 设计概述

### 1.1 设计目标

1. 界面简洁直观，降低图书管理员的学习成本
2. 高效的操作流程，支持扫码枪和键盘快捷键操作
3. 清晰的视觉层级，引导用户聚焦当前任务
4. 响应式布局，PC 端全功能，平板端适配核心操作

### 1.2 布局规范

- **管理后台**：左侧导航栏 (256px) + 右侧内容区
- **读者端**：顶部导航栏 + 内容区
- **表单**：标签宽度 100px，输入框宽度 320px（标准）/ 100% (响应式)
- **表格**：支持分页（默认每页 20 条）、排序、行选中
- **弹窗**：宽度 520px（标准表单）/ 720px（详情）
- **间距**：卡片内边距 20px，组件间距 16px，段落间距 12px

### 1.3 交互规范

- 不可逆操作（删除、下架）需二次确认弹窗
- 表单提交后显示 loading 状态，成功后显示绿色提示，失败显示红色提示
- 列表页为空时展示空状态占位图
- 顶部导航固定在视口顶部，内容区超出时内部滚动
- 关键操作使用确认弹窗（el-message-box），一般操作使用消息提示（el-message）

## 4. 交互状态覆盖

### 4.1 加载状态

| 场景 | 表现 |
|------|------|
| 页面初始加载 | 骨架屏或 el-skeleton |
| 列表数据加载 | 表格 loading 遮罩 + 转圈动画 |
| 表单提交 | 按钮 loading + 禁用状态 |
| 图片加载 | 封面图加载前显示灰色占位图 |

### 4.2 空状态

| 场景 | 表现 |
|------|------|
| 搜索无结果 | 空状态插图 + "未找到匹配的图书" |
| 借阅列表为空 | 空状态插图 + "暂无借阅记录" |
| 逾期列表为空 | 空状态插图 + "当前没有逾期记录" |
| 操作日志为空 | 空状态插图 + "暂无操作日志" |

### 4.3 错误状态

| 场景 | 表现 |
|------|------|
| 网络错误 | API 请求失败 → 页面顶部红色 bar 或 el-message 错误提示 |
| 表单校验失败 | 输入框下方红色提示文字，输入框红色边框 |
| 权限不足 | 前端路由守卫拦截 → 跳转 403 页或首页 |
| Token 过期 | Axios 拦截器检测 401 → 清除 Token → 跳转登录页 |
| 500 错误 | 统一错误页 "服务器开小差了，请稍后重试" |

### 4.4 确认操作

| 操作 | 确认文案 |
|------|---------|
| 删除用户 | "确定要删除用户 [用户名] 吗？删除后不可恢复。" |
| 下架图书 | "确定要下架《书名》吗？下架后不可被借阅。" |
| 恢复默认配置 | "确认恢复所有配置项为默认值？" |
| 确认借出 | "确认借出 N 本图书？" |
| 确认归还 | "确认归还《书名》？"（逾期时附罚款信息） |

## 5. 响应式适配

### 5.1 断点定义

| 断点 | 宽度 | 布局 |
|------|------|------|
| PC | ≥ 1200px | 全部功能，管理后台侧边栏+内容区 |
| 平板 | 768px ~ 1199px | 管理后台侧边栏折叠，表格横向滚动 |
| 手机 | < 768px | 仅图书检索浏览功能，管理后台不可用 |

### 5.2 平板适配要点

- 管理后台左侧导航默认折叠（仅显示图标）
- 图书卡片 5 列 → 3 列
- 表格列过多时出现横向滚动条
- 借书/还书页面改为上下布局（取消左右分栏）
- 表单标签改为顶部对齐
