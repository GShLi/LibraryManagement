## 1. API 概述

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| RESTful | 使用 HTTP 标准方法（GET/POST/PUT/DELETE），资源名使用名词复数 |
| JSON | 所有请求和响应使用 `Content-Type: application/json` |
| 认证 | 除注册/登录外，所有 API 需在 Header 中携带 `Authorization: Bearer <token>` |
| 分页 | 列表接口统一使用 `?page=1&pageSize=20`，响应含 `total`, `page`, `pageSize`, `list` |
| 版本 | API 路径不含版本号，版本变更通过 Header `Accept-Version` 或新路径协商 |
| 幂等 | GET/PUT/DELETE 操作保证幂等，POST 操作不保证 |

### 1.2 基础 URL

```
开发环境: http://localhost:3000
生产环境: https://<domain> (由部署配置决定)
```

### 1.3 统一响应格式

#### 成功响应

```json
{
  "code": 200,
  "message": "ok",
  "data": { ... }
}
```

分页列表响应：

```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "total": 150,
    "page": 1,
    "pageSize": 20,
    "list": [ ... ]
  }
}
```

#### 错误响应

```json
{
  "code": 400,
  "message": "参数校验失败",
  "data": null
}
```

字段级验证错误：

```json
{
  "code": 422,
  "message": "请求参数校验失败",
  "data": {
    "errors": [
      { "field": "username", "message": "用户名不能为空" },
      { "field": "password", "message": "密码长度至少 8 位" }
    ]
  }
}
```

## 2. 错误码定义

### 2.1 HTTP 状态码

| 状态码 | 含义 | 使用场景 |
|--------|------|---------|
| 200 | OK | GET/PUT/DELETE 操作成功 |
| 201 | Created | POST 创建资源成功 |
| 204 | No Content | DELETE 操作成功（无响应体） |
| 400 | Bad Request | 请求参数格式错误 |
| 401 | Unauthorized | 未认证或 Token 无效/过期 |
| 403 | Forbidden | 已认证但权限不足 |
| 404 | Not Found | 资源不存在 |
| 409 | Conflict | 资源冲突（唯一约束、状态冲突） |
| 422 | Unprocessable Entity | 参数校验失败（业务规则） |
| 429 | Too Many Requests | 请求频率超限 |
| 500 | Internal Server Error | 服务器内部错误 |

## 3. 认证方案

### 3.1 JWT Token

| 属性 | 说明 |
|------|------|
| 签发 | 用户登录成功后由服务端签发 |
| 载荷 | `{ "userId": 1, "username": "admin", "role": "admin", "iat": ..., "exp": ... }` |
| 有效期 | 默认 24 小时；"记住我" 勾选则为 7 天 |
| 算法 | HS256，密钥由环境变量 `JWT_SECRET` 配置 |
| 传输 | 前端存储于 localStorage，通过 Axios 拦截器自动附加到每个请求的 Authorization Header |
| 刷新 | Token 过期需重新登录；不提供独立的 refresh token |

### 3.2 认证流程

```
1. 用户登录 → POST /api/auth/login → 返回 { token, user }
2. 前端存储 token → localStorage.setItem('token', token)
3. 每次请求 → Axios 拦截器自动添加 Header: Authorization: Bearer <token>
4. 后端认证中间件 → 解析 JWT → 校验签名和有效期 → 注入 req.user
5. Token 过期 → 后端返回 401 → 前端拦截器清除 token 并跳转登录页
```

### 3.3 权限角色

| 角色 | 标识 | 权限范围 |
|------|------|---------|
| 系统管理员 | admin | 全部接口、系统配置、用户管理、日志审计、数据备份 |
| 图书管理员 | librarian | 图书管理、借阅管理、读者管理、统计查询 |
| 普通读者 | reader | 检索图书、查看个人中心、续借(个人)、预约(个人)、借阅历史(个人) |
| 游客 | — | 仅图书检索浏览（无需 Token） |

### 3.4 权限校验中间件

```javascript
// authMiddleware — 解析 JWT，注入 req.user
// requireAuth — 检查 req.user 是否存在（已登录）
// requireRole(...roles) — 检查 req.user.role 是否在允许列表中
```

## 4. 接口详细定义

### 4.1 认证模块 `/api/auth`

#### 4.1.1 用户注册

| 属性 | 内容 |
|------|------|
| 路径 | `/api/auth/register` |
| 方法 | POST |
| 认证 | 否 |

**请求参数 (Body)**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 登录用户名，3-50 字符 |
| password | string | 是 | 密码，8-32 位，必须含字母和数字 |
| name | string | 是 | 真实姓名 |
| phone | string | 是 | 手机号，1 开头 11 位数字 |
| email | string | 否 | 电子邮箱 |

**请求示例**

```json
{
  "username": "zhangsan",
  "password": "abc12345",
  "name": "张三",
  "phone": "13800138000",
  "email": "zhangsan@example.com"
}
```

**成功响应 (201)**

```json
{
  "code": 201,
  "message": "注册成功",
  "data": {
    "userId": 1,
    "username": "zhangsan",
    "readerNo": "R-20260704-00001"
  }
}
```

**错误响应**

```json
{ "code": 40111, "message": "用户名已被注册", "data": null }
{ "code": 40112, "message": "手机号已被注册", "data": null }
{ "code": 40113, "message": "密码强度不足，需至少 8 位且包含字母和数字", "data": null }
{ "code": 422, "message": "请求参数校验失败", "data": { "errors": [...] } }
```

**说明**：注册后自动创建 users 记录和 readers 记录，角色默认为 "reader"，读者状态 "active"。生成唯一读者证号格式 `R-YYYYMMDD-XXXXX`。

#### 4.1.2 用户登录

| 属性 | 内容 |
|------|------|
| 路径 | `/api/auth/login` |
| 方法 | POST |
| 认证 | 否 |

**请求参数 (Body)**

| 参数 | 类型 | 必填 | 说明 |

**请求示例**

```json
{
  "username": "zhangsan",
  "password": "abc12345",
  "rememberMe": true
}
```

**成功响应 (200)**

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": 1,
      "username": "zhangsan",
      "role": "reader",
      "name": "张三",
      "readerNo": "R-20260704-00001"
    }
  }
}
```

**错误响应**

```json
{ "code": 40110, "message": "用户名或密码错误", "data": null }
{ "code": 40114, "message": "账号已被禁用", "data": null }
{ "code": 40115, "message": "账号已临时锁定，请 15 分钟后重试", "data": null }
```

**说明**：
- 连续登录失败 ≥ 5 次，账号锁定 15 分钟
- "记住我"勾选时 Token 有效期 7 天，否则 24 小时
- 登录成功后记录 audit_log

| 属性 | 内容 |
|------|------|
| 路径 | `/api/borrows` |
| 方法 | POST |
| 权限 | admin / librarian |

**请求参数 (Body)**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| readerNo | string | 是 | 读者证号或手机号 |
| barcodes | string[] | 是 | 图书条码数组，支持批量借阅 |
| borrowDays | integer | 否 | 借阅天数，默认取系统配置 `borrow_duration_days` |

**请求示例**

```json
{
  "readerNo": "R-20260704-00001",
  "barcodes": ["BC-20260704-00001", "BC-20260704-00002"]
}
```

**成功响应 (201)**

```json
{
  "code": 201,
  "message": "借书成功",
  "data": {
    "borrowRecords": [
      {
        "borrowId": 101,
        "barcode": "BC-20260704-00001",
        "bookTitle": "JavaScript 高级程序设计（第4版）",
        "dueDate": "2026-08-03"
      },
      {
        "borrowId": 102,
        "barcode": "BC-20260704-00002",
        "bookTitle": "深入浅出 Node.js",
        "dueDate": "2026-08-03"
      }
    ]
  }
}
```

**错误响应**

```json
{ "code": 40301, "message": "读者不存在", "data": null }
{ "code": 40302, "message": "读者账户已冻结", "data": null }
{ "code": 40303, "message": "读者账户已挂失", "data": null }
{ "code": 40304, "message": "读者账户已禁用", "data": null }
{ "code": 40502, "message": "读者已达借阅上限", "data": null }
{ "code": 40503, "message": "读者有逾期未还图书", "data": null }
{ "code": 40504, "message": "读者有未缴纳罚款", "data": null }
{ "code": 40408, "message": "该副本不可借", "data": null }
{ "code": 40505, "message": "该图书已被借出", "data": null }
{ "code": 40506, "message": "该图书已被其他读者预约", "data": null }
```

**说明**：
- 逐本校验读者资格和副本状态
- 事务操作：批量创建 borrow 记录 + 更新 book_copy 状态 + 更新 reader.current_borrowed + 更新 book.available_copies
- 若有预约且借书人是预约者，自动 fulfill 预约
- 并发冲突时事务回滚

```json
  "data": {
    "borrowId": 101,
    "returnDate": "2026-07-10T14:20:00.000Z",
    "overdue": false,
    "fine": null
  }
}
```

**逾期归还响应**

```json
{
  "code": 200,
  "message": "还书成功，该图书已逾期，请缴纳罚款",
  "data": {
    "borrowId": 102,
    "returnDate": "2026-07-10T14:20:00.000Z",
    "overdue": true,
    "overdueDays": 5,
    "fine": {
      "fineId": 15,
      "amount": 0.50,
      "status": "unpaid"
    }
  }
}
```

**错误响应**

```json
{ "code": 40501, "message": "借阅记录不存在", "data": null }
{ "code": 40507, "message": "该记录已归还", "data": null }
```

**说明**：
- 还书时计算逾期：actual_return_date > due_date 则逾期
- 罚款金额 = overdue_days × 0.1，上限为定价 × 2
- 逾期 ≥ 30 天自动冻结读者账户
- 事务操作：更新 borrow + 更新 book_copy 状态 + 更新 reader.current_borrowed + 更新 book.available_copies + 可能创建 fine
