## 1. 性能设计

### 1.1 性能指标

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| 首屏加载时间 (FCP) | ≤ 2 秒 | Lighthouse 测试，3G 网络模拟 |
| 页面切换时间 | ≤ 300ms | Vue Router 导航钩子计时 |
| API 普通查询响应 | ≤ 500ms (P95) | 服务端 Winston 日志统计 |
| API 复杂统计响应 | ≤ 2s (P95) | 服务端 Winston 日志统计 |
| API 列表查询响应 | ≤ 300ms (P95) | 服务端 Winston 日志统计 |
| 借书操作响应 | ≤ 1s | 含事务提交时间 |
| 批量导入 5000 条 | ≤ 30s | 前端计时 + 后端日志 |
| 并发用户数 | 50 在线 | 压力测试工具（Artillery / k6） |
| 数据库写入吞吐 | 50 TPS | 事务性操作压测 |

### 1.2 前端性能优化

| 策略 | 实现方式 |
|------|---------|
| **代码分割** | Vue Router 路由级懒加载，`() => import('./views/xxx.vue')` |
| **静态资源** | Vite 构建自动代码分割，vendor chunk 分离（vue, element-plus, axios） |
| **图片优化** | 封面图片压缩（≤ 200KB），懒加载 |
| **组件优化** | Element Plus 按需导入（unplugin-vue-components），减小打包体积 |
| **缓存策略** | 分类数据、系统配置等低频变更数据前端缓存（Pinia + localStorage），5 分钟过期 |
| **虚拟滚动** | 长列表（如审计日志）使用 Element Plus 虚拟化表格 |
| **打包优化** | Vite Rollup 配置 gzip 压缩，生产构建 ≤ 500KB (gzip) |

### 1.3 后端性能优化

| 策略 | 实现方式 |
|------|---------|
| **数据库连接** | SQLite WAL 模式，允许并发读 |
| **查询优化** | 分页查询使用 LIMIT/OFFSET；索引覆盖高频查询条件 |
| **统计缓存** | 概览统计数据缓存 5 分钟，避免每次请求实时计算 |
| **批量操作** | 批量入库使用 Sequelize `bulkCreate`，事务包裹 |
| **日志异步** | Winston 文件 transport 异步写，不阻塞主线程 |
| **静态文件** | 生产环境由 Nginx 直接返回静态资源，不经过 Express |

### 1.4 SQLite 性能调优

```sql
-- WAL 模式（提升并发写入）
PRAGMA journal_mode = WAL;

-- 内存缓存大小（默认 2MB → 推荐 64MB，视服务器内存调整）
PRAGMA cache_size = -64000;

-- 临时文件存内存（减少磁盘 I/O）
PRAGMA temp_store = MEMORY;

-- 同步模式（NORMAL 平衡性能与安全）
PRAGMA synchronous = NORMAL;

-- 外键约束
PRAGMA foreign_keys = ON;
```

## 2. 安全设计

### 2.1 安全目标

| 维度 | 目标 |
|------|------|
| 机密性 | 用户密码不可还原；敏感数据不泄露 |
| 完整性 | 所有输入经过校验；数据操作遵循业务规则 |
| 可用性 | 防御常见 Web 攻击（注入、XSS、CSRF、暴力破解） |
| 可审计 | 关键操作全程记录，日志不可篡改删除 |

### 2.2 认证安全

| 措施 | 实现 |
|------|------|
| 密码加密 | bcryptjs，salt 轮数 10 |
| 密码强度 | 注册和修改时强制校验：≥ 8 位，含字母和数字 |
| Token 签名 | HS256（HMAC-SHA256），密钥 ≥ 256 位随机字符串 |
| Token 有效期 | 默认 24h，"记住我" 7 天 |
| Token 刷新 | 不支持 Token 刷新（过期须重新登录），避免 Refresh Token 被窃取 |
| 失败锁定 | 同一账号连续失败 5 次，锁定 15 分钟 |
| 旧密码校验 | 修改密码时须验证旧密码 |

### 2.3 授权安全 (RBAC)

| 角色 | 权限范围 |
|------|---------|
| admin | 全部功能：用户管理、系统配置、日志审计、数据备份，以及 librarian 所有权限 |
| librarian | 图书管理（入库/编目/下架）、借阅管理（借书/还书/罚款）、读者管理（查看/编辑） |
| reader | 图书检索（完整）、续借（自己的）、预约（自己的）、个人中心、借阅历史（自己的） |
| 游客 | 图书检索（仅可借和借出状态的书目信息） |

**权限校验流程**：
```
JWT 认证 → 提取 req.user.role → 匹配路由权限表 → 放行 / 403
```

**权限变更即时生效**：角色权限映射存储在 system_config 表或代码常量中，JWT 仅存角色名，不存权限列表。权限变更后，用户下次 API 请求时重新查库校验。

### 2.4 输入安全

| 威胁 | 防御措施 |
|------|---------|
| SQL 注入 | Sequelize ORM 参数化查询，禁止拼接 SQL |
| XSS | 输入过滤：后端对字符串字段转义 HTML 实体；前端 Vue 模板自动转义 |
| CSRF | SPA + JWT Bearer Token 认证模式下，Token 不通过 Cookie 传递，天然防御 CSRF |
| 文件上传 | 仅允许图片格式（jpg/png/gif/webp），大小 ≤ 2MB；校验文件魔数 |
| 参数篡改 | 前后端双重校验：前端表单规则 + 后端 express-validator |
| 路径遍历 | 文件路径严格限制在 uploads/ 和 backup/ 目录内 |

### 2.5 HTTP 安全头 (Helmet)

| Header | 值 |
|--------|-----|
| Content-Security-Policy | default-src 'self'; img-src 'self' data:; style-src 'self' 'unsafe-inline' |
| X-Frame-Options | DENY |
| X-Content-Type-Options | nosniff |
| X-XSS-Protection | 0 (禁用旧版过滤器，依赖 CSP) |
| Strict-Transport-Security | max-age=31536000; includeSubDomains |
| Referrer-Policy | strict-origin-when-cross-origin |

### 2.6 速率限制

| 端点 | 限制 | 说明 |
|------|------|------|
| POST /api/auth/login | 10 次/分钟/IP | 防暴力破解 |
| POST /api/auth/register | 3 次/分钟/IP | 防批量注册 |
| 其他所有端点 | 100 次/分钟/用户 | 防 API 滥用 |

实现：express-rate-limit 中间件。

## 3. 可用性设计

### 3.1 可用性指标

| 指标 | 目标值 |
|------|--------|
| 系统可用率 (SLA) | 99.5%（月度允许停机 ≤ 3.6 小时） |
| 计划内维护窗口 | 每月 ≤ 2 小时 |
| 故障恢复时间 (RTO) | ≤ 4 小时 |
| 数据恢复点 (RPO) | ≤ 24 小时（依赖每日备份） |

### 3.2 高可用策略

| 策略 | 实现 |
|------|------|
| 进程守护 | pm2 进程管理，崩溃自动重启 |
| 多进程 | pm2 Cluster 模式，利用多核 CPU（实例数 = CPU 核数） |
| 零停机部署 | pm2 reload 平滑重启，先启动新进程再关闭旧进程 |
| 健康检查 | GET /api/health 返回 `{ status: "ok", uptime, dbConnected }`，Nginx 可配置健康探测 |
| 优雅关闭 | Express 监听 SIGTERM，先关闭 HTTP 服务再断开数据库连接 |

### 3.3 容错设计

| 场景 | 处理策略 |
|------|---------|
| 数据库操作失败 | 事务回滚，返回 500，记录错误日志 |
| 并发借出同一副本 | 数据库事务 + 行级锁（SQLite 自动处理），冲突方返回 409 |
| 第三方 ISBN 接口超时 | 5 秒超时降级，允许手动录入 |
| 请求超时 | 设置 30 秒全局超时（express-timeout），超时返回 504 |
| 未捕获异常 | process.on('uncaughtException') 捕获，记录日志，优雅退出后 pm2 重启 |

### 3.4 数据备份与恢复

| 项目 | 策略 |
|------|------|
| 备份方式 | 复制 SQLite 数据库文件（文件锁保证一致性） |
| 备份频率 | 每日凌晨 2:00 自动备份 |
| 备份保留 | 最近 7 天滚动保留 |
| 备份命名 | `library_YYYYMMDD_HHmmss.db` |
| 备份存储 | 本地 `server/backup/` 目录 |
| 恢复流程 | 1. 暂停服务 2. 替换 .db 文件 3. 重启服务 |
| 验证 | 每月至少一次备份恢复演练 |

### 3.5 错误处理

**前端**：
- Axios 响应拦截器统一处理错误：401 → 跳转登录页；403 → 提示无权限；500 → 统一提示
- Element Plus Message 组件展示友好的中文错误提示
- 网络异常/超时给出"请检查网络连接"提示

**后端**：
- 全局 errorHandler 中间件捕获所有异常
- 区分已知业务异常（自定义错误码）和未知系统异常（返回通用 500）
- 仅在开发环境返回堆栈信息（`NODE_ENV=development`）

## 4. 可扩展性设计

### 4.1 水平扩展能力

虽然当前设计为单体部署，但架构预留了扩展点：

| 扩展点 | 当前方案 | 扩展方向 |
|--------|---------|---------|
| 数据库 | SQLite | 替换为 PostgreSQL/MySQL（仅需修改 Sequelize 配置） |
| 会话 | JWT 无状态 | 天然支持多实例扩展，无需会话同步 |
| 文件存储 | 本地磁盘 | 接入 OSS/S3 对象存储 |
| 搜索引擎 | SQL LIKE 模糊搜索 | 接入 Elasticsearch 全文检索 |
| 缓存 | 内存变量 + localStorage | 接入 Redis 集中缓存 |

### 4.2 模块解耦

| 层面 | 设计 |
|------|------|
| Service 层 | 按业务领域独立划分，不相互调用（如 BorrowService 不直接依赖 FineService，由 Controller 编排） |
| 共享逻辑 | 提取到 utils/ 目录（条码生成、日期计算、校验规则等） |
| 配置外置 | 所有可变参数通过 system_config 表或 .env 环境变量管理 |

### 4.3 API 版本管理

- 当前版本：`v1`（URL 前缀 `/api/v1/`，初期可简化为 `/api/`）
- 未来升级：新增 `/api/v2/` 路径，旧版本保留至所有客户端迁移完成
- 版本生命周期：v1 在 v2 发布后保留 6 个月

## 5. 可维护性设计

### 5.1 代码规范

| 工具 | 用途 |
|------|------|
| ESLint | JavaScript/TypeScript 规范检查，使用 recommended 规则集 |
| Prettier | 代码格式化（统一缩进、引号、分号等风格） |
| TypeScript | 前端强制使用 TS，后端可选（关键模块建议使用） |

### 5.2 日志规范

| 级别 | 用途 | 示例 |
|------|------|------|
| error | 系统错误、异常 | 数据库连接失败、第三方接口超时 |
| warn | 业务警告 | 登录失败、权限拒绝、数量超限 |
| info | 关键操作 | 借书成功、还书成功、用户创建 |
| debug | 调试信息 | SQL 查询语句、请求参数详情 |

**日志格式**（结构化 JSON）：
```json
{
  "level": "info",
  "timestamp": "2026-07-04T10:30:00.000Z",
  "message": "borrow.create 成功",
  "userId": 3,
  "action": "borrow.create",
  "targetType": "borrow",
  "targetId": "42",
  "ip": "192.168.1.100"
}
```

**日志存储**：
- `logs/app-%DATE%.log` — 按日滚动
- `logs/error-%DATE%.log` — 仅 error 级别
- 保留 30 天

### 5.3 配置管理

| 配置类型 | 管理方式 | 示例 |
|---------|---------|------|
| 环境配置 | `.env` 文件 | DB_PATH, JWT_SECRET, PORT, NODE_ENV |
| 业务参数 | `system_config` 表 | 借阅天数、罚款费率、预约保留天数 |
| 前端常量 | `config.ts` 文件 | API_BASE_URL, TOKEN_KEY, PAGE_SIZE |

配置优先级：环境变量 > system_config 表 > 代码默认值

### 5.4 监控指标

| 指标 | 采集方式 |
|------|---------|
| 请求量/成功率/响应时间 | Winston 日志 + 请求中间件统计 |
| 错误率 | error 日志计数 |
| 在线用户数 | WebSocket（可选，初期不实现） |
| 数据库大小 | 文件系统 du 命令 |
| 进程状态 | pm2 monit / pm2 list |

## 7. 可靠性设计

### 7.1 数据一致性

| 操作 | 事务范围 | 隔离级别 |
|------|---------|---------|
| 借书 | borrow 写入 + book_copy 状态更新 + readers 计数更新 + books.copies 更新 | SERIALIZABLE |
| 还书 | borrow 状态更新 + book_copy 状态更新 + readers 计数更新 + fine 写入（如逾期） | SERIALIZABLE |
| 续借 | borrow 记录更新 | SERIALIZABLE |
| 批量入库 | books 写入 + book_copy 批量写入 | SERIALIZABLE |
| 系统配置 | system_config 批量更新 | SERIALIZABLE |

SQLite 的 SERIALIZABLE 隔离级别通过数据库级锁实现，天然保证串行化。

### 7.2 冗余字段维护

| 表 | 字段 | 更新触发条件 |
|----|------|------------|
| books.total_copies | 总副本数 | 入库/下架/重新上架 |
| books.available_copies | 可借副本数 | 借书/还书/下架/重新上架 |
| readers.current_borrowed | 当前借阅数 | 借书(+1)/还书(-1) |

冗余字段在事务中同步更新，保证与 book_copy 表数据一致。

### 7.3 定时任务

| 任务 | 频率 | 实现 |
|------|------|------|
| 逾期检测 | 每日 00:00 | node-cron 或系统 cron 调用脚本 |
| 数据库备份 | 每日 02:00 | 系统 cron + shell 脚本 |
| 日志轮转 | 每日 00:00 | Winston DailyRotateFile |
| 过期预约清理 | 每日 01:00 | 将过期预约状态更新为 expired |
