## 1. 架构概述

### 1.1 架构目标

- **前后端分离**：前端 SPA 与后端 RESTful API 独立开发、独立部署、独立演进
- **分层解耦**：后端采用经典三层架构（Controller → Service → DAO），每层职责单一
- **轻量级部署**：基于 SQLite 的单体应用，零依赖数据库服务，适合中小型图书馆
- **安全可审计**：JWT 认证 + RBAC 授权 + 操作日志全覆盖

### 1.3 架构决策记录 (ADR)

| 决策编号 | 决策 | 理由 | 替代方案 |
|---------|------|------|---------|
| ADR-001 | 前后端分离架构 | 前端 SPA 与后端 API 独立开发部署，团队可并行工作；后端 API 可被第三方集成 | 服务端渲染(SSR) — 但 SPA 用户体验更好，适合管理系统类应用 |
| ADR-002 | SQLite 数据库 | 嵌入式零配置，无需独立数据库服务；单文件备份恢复简单；满足 ≤5 万读者 + 10 万图书规模 | MySQL / PostgreSQL — 适合更大规模场景，但运维复杂度显著增加 |
| ADR-003 | Express 框架 | Node.js 生态最成熟的 Web 框架，社区资源丰富，中间件机制灵活 | Koa / Fastify — 更现代但生态不如 Express 丰富 |
| ADR-004 | Sequelize ORM | 成熟的 SQLite 支持，声明式模型定义，自动建表迁移，事务支持完善 | Knex.js / Prisma — Prisma 对 SQLite 支持有限，Knex 非完整 ORM |
| ADR-005 | JWT 无状态认证 | 无需服务端存储会话，天然支持 API 无状态；RBAC 角色信息内嵌 Token | Session-based — 需要服务端存储，不利于 RESTful API 扩展 |
| ADR-006 | JWT 中不存储权限详情 | Token 仅含用户 ID 和角色名(core identity)，权限校验实时查询数据库 | 在 Token 中嵌入完整权限列表 — Token 体积膨胀且权限变更不即时生效 |

## 2. 技术选型

### 2.1 前端技术栈

| 技术 | 版本 | 选型理由 |
|------|------|---------|
| Vue 3 | ≥ 3.3 | 渐进式框架，组合式 API(Composition API)提升代码复用和逻辑组织；TypeScript 支持良好；虚拟 DOM 性能优异 |
| Element Plus | ≥ 2.3 | 专为 Vue 3 设计的企业级 UI 组件库；提供表格、表单、弹窗、导航等管理系统常用组件；中文文档完善 |
| Vite | ≥ 4.0 | 基于 ESM 的极速构建工具；开发环境秒级 HMR；生产环境 Rollup 打包 |
| Vue Router | 4.x | Vue 3 官方路由，支持嵌套路由、路由守卫、动态路由；配合 JWT 实现路由级权限控制 |
| Pinia | 2.x | Vue 3 官方状态管理库；TypeScript 友好；模块化 Store 设计 |
| Axios | 1.x | 基于 Promise 的 HTTP 客户端；请求/响应拦截器统一处理 Token 注入和错误处理 |
| TypeScript | ≥ 5.0 | 静态类型检查，提升代码可维护性 |

### 2.2 后端技术栈

| 技术 | 版本 | 选型理由 |
|------|------|---------|
| Node.js | ≥ 18 LTS | 事件驱动非阻塞 I/O，适合 I/O 密集型 Web 应用；前后端 JavaScript 统一技术栈 |
| Express | ≥ 4.18 | Node.js 生态最成熟的 Web 框架；中间件机制灵活；社区中间件丰富 (helmet, cors, morgan 等) |
| Sequelize | ≥ 6.0 | 支持 SQLite 的成熟 ORM；声明式模型定义 + 自动建表；内置事务、关联查询、钩子 |
| SQLite 3 | ≥ 3.35 | 嵌入式关系型数据库；零配置零运维；单文件便于备份迁移；满足项目数据容量需求(≤300MB) |
| jsonwebtoken | ≥ 9.0 | JWT 签发与校验；支持 RS256/HS256 算法 |
| bcryptjs | ≥ 2.0 | 纯 JavaScript bcrypt 实现，跨平台无编译依赖 |
| Winston | ≥ 3.0 | 结构化日志；支持多 transport(文件/控制台)；按日志级别过滤 |
| express-validator | ≥ 7.0 | 请求参数校验中间件；声明式校验规则 |

### 4.3 部署方案说明

| 组件 | 说明 |
|------|------|
| Nginx | 反向代理 + 静态资源服务；SSL 终结；gzip 压缩；可配置限流和 IP 白名单 |
| pm2 Cluster | Node.js 多进程管理；利用多核 CPU；自动重启崩溃进程；零停机重载(reload) |
| SQLite | 单文件数据库，部署时无需额外安装；WAL 模式提升并发读性能 |
| cron | 系统级定时任务；逾期检测脚本 + 数据库备份脚本 |

## 5. 安全设计

### 5.1 认证与授权

```
┌──────────┐      ┌──────────┐      ┌──────────┐
│  客户端   │      │  Express  │      │  SQLite   │
└─────┬────┘      └─────┬────┘      └─────┬────┘
      │                 │                 │
      │ POST /auth/login│                 │
      │────────────────▶│                 │
      │                 │ 查询用户         │
      │                 │────────────────▶│
      │                 │ 用户信息         │
      │                 │◀────────────────│
      │                 │                 │
      │                 │ bcrypt 比对密码  │
      │                 │ 检查锁定状态     │
      │                 │ 更新失败计数     │
      │                 │                 │
      │ JWT Token       │ 签发 JWT:       │
      │◀────────────────│ {id, username,  │
      │                 │  role, iat, exp}│
      │                 │                 │
      │ API 请求 + Token │                 │
      │────────────────▶│                 │
      │                 │ 验证签名 + 有效期 │
      │                 │ 提取 req.user    │
      │                 │ RBAC 校验角色     │
      │                 │                 │
```

### 5.2 安全措施清单

| 层级 | 措施 | 说明 |
|------|------|------|
| 传输 | HTTPS | 生产环境强制 HTTPS，Nginx SSL 终结 |
| 认证 | JWT + bcrypt | 密码 salt 10 轮哈希；Token 有效期 24h（记住我 7d） |
| 授权 | RBAC | 角色→权限映射，中间件校验；权限变更即时生效 |
| 输入 | 双重校验 | 前端 Element Plus 表单校验 + 后端 express-validator |
| 输出 | Helmet | 设置安全 HTTP 头（CSP, X-Frame-Options, X-Content-Type-Options 等） |
| 跨域 | CORS | 白名单 Origin，严格限制允许的 Method 和 Header |
| 注入 | Sequelize 参数化 | ORM 使用参数化查询，防止 SQL 注入 |
| 攻击 | 速率限制 | express-rate-limit 限制 API 调用频率（登录端点额外限制） |
| 日志 | 审计不可删除 | audit_log 表无 DELETE 接口；日志不含密码明文 |
| 锁定 | 登录失败锁定 | 5 次失败锁定 15 分钟 |

### 6.2 借书事务流程

```
POST /api/borrows { reader_id, barcodes: [...] }
     │
     ▼
Controller: borrow.controller.js → createBorrow()
     │
     ▼
Service: borrow.service.js → borrowBooks()
     │
     ├─ 1. 查询读者状态（readers 表）
     │     ├─ 不存在 → throw NotFoundError
     │     ├─ 冻结/禁用 → throw ForbiddenError
     │     ├─ 有逾期未还 → throw ConflictError
     │     ├─ 借阅数达上限 → throw ConflictError
     │     └─ 通过 ✓
     │
     ├─ 2. 逐本校验（book_copy 表）
     │     ├─ 条码不存在 → throw NotFoundError
     │     ├─ 非"可借"状态 → throw ConflictError
     │     └─ 通过 ✓
     │
     ├─ 3. 开启数据库事务
     │     ├─ 创建 borrow 记录（每条）
     │     ├─ 更新 book_copy.status = "borrowed"
     │     ├─ 更新 readers.current_borrowed += N
     │     ├─ 更新 books.available_copies -= N
     │     └─ 记录 audit_log
     │
     ├─ 提交事务 ✓
     │
     └─ 返回借阅结果列表（含应还日期）
```
